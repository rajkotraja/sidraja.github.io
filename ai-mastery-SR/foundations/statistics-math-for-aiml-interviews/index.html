Core Statistical Concepts (Foundations for AI Models)

What is the mean, and how is it used in model evaluation or dataset analysis?

The mean is the average value of a dataset. It is calculated by adding all values and dividing by the total number of values.

Formula

  • Mean = (sum of all values) / (number of values)

Example (dataset analysis)

Suppose you have the number of hours users spend daily on an app:

  • [2, 3, 3, 4, 8]

Mean = (2 + 3 + 3 + 4 + 8) / 5 = 4

This tells you that, on average, users spend 4 hours per day.

Use in ML

  • Used to summarize feature values (e.g., average age, average price)
  • Used in normalization/standardization
  • Used in evaluation metrics (e.g., Mean Squared Error uses mean of squared errors)

Example (model evaluation)

If prediction errors are:

  • [1, 2, 3, 4]

Mean error = (1 + 2 + 3 + 4) / 4 = 2.5

This represents the average prediction error of the model.


When is the median a better measure than the mean in real-world ML datasets?

The median is the middle value when data is sorted.

Median is better when the data contains outliers or is skewed.

Example

User salaries in a dataset:

  • [20k, 25k, 30k, 35k, 500k]

Mean = (20 + 25 + 30 + 35 + 500) / 5 = 122k

Median = 30k

Most users earn around 30k, but one very high value pulls the mean upward.

Median better represents a typical user.

Use in ML

  • Feature analysis for skewed variables (salary, income, purchases)
  • Used in robust scaling
  • Better indicator of central tendency when outliers exist

What is the mode, and where does it appear in classification problems?

The mode is the value that appears most frequently in a dataset.

Example

User device types:

  • ["mobile", "mobile", "desktop", "mobile", "tablet"]

Mode = "mobile"

Use in ML classification

  • Represents the most common class
  • Used as a baseline model (majority class predictor)
  • Helps identify class imbalance

Example

Labels in a dataset:

  • [spam, not_spam, spam, spam, spam]

Mode = spam

A naive classifier could predict “spam” for every input and still achieve decent accuracy if the data is imbalanced.


How do mean, median, and mode behave in skewed data commonly found in user-generated datasets?

User-generated data (clicks, likes, views, time spent) is often skewed, not symmetric.

Right-skewed data example

Time spent on a page (minutes):

  • [1, 2, 2, 3, 4, 30]
  • Mean = pulls toward 30 → 7
  • Median = middle value → 2.5
  • Mode = most frequent → 2

Behavior

  • Mean shifts toward extreme values
  • Median stays stable
  • Mode shows most common behavior

Typical relationship

  • Right skew: Mean > Median > Mode
  • Left skew: Mean < Median < Mode

In ML, median or mode often gives a better picture of user behavior than mean.


What does variance represent in a dataset used for training ML models?

Variance measures how much data points spread out from the mean.

It answers:

“How different are the values from the average?”

Formula idea

  • Variance = average of squared differences from the mean

Example

Feature values:

  • [10, 10, 10, 10] → variance = 0
  • [5, 10, 15, 20] → variance = high

Use in ML

  • Indicates feature variability
  • Helps detect noisy or unstable features
  • Connected to the bias–variance tradeoff

Low variance → data points are similar

High variance → data points vary widely


How is standard deviation interpreted in the context of feature distribution?

Standard deviation (SD) is the square root of variance.

It shows how far values typically deviate from the mean.

Example

Heights of users (cm):

  • Mean = 170
  • SD = 5

Most users lie between:

  • 165 and 175

In feature distribution

  • Small SD → feature values tightly clustered
  • Large SD → feature values widely spread

ML usage

  • Used in standardization:

    (value − mean) / standard deviation

  • Helps compare spread across different features


Why is standard deviation more commonly used than variance in ML analysis?

Variance uses squared units, which are hard to interpret.

Example

  • Height variance = 25 (cm²)
  • Height SD = 5 (cm)

Standard deviation:

  • Uses the same unit as the feature
  • Is easier to understand and communicate
  • Is directly usable in scaling and thresholds

Most ML preprocessing techniques work with standard deviation, not variance.


How does high variance in data affect model training and generalization?

High variance in data means:

  • Large spread
  • High noise
  • Many extreme values

Impact on training

  • Model may try to fit noise
  • Leads to overfitting
  • Sensitive to small data changes

Example

If user behavior data varies wildly:

  • Model memorizes specific users
  • Performs well on training data
  • Fails on new users

Practical effects

  • Unstable predictions
  • Poor generalization
  • Higher error on unseen data

Common solutions

  • Feature scaling
  • Outlier handling
  • Regularization
  • Collecting more representative data

High variance in data often translates into high variance in the model if not controlled.


How is probability formally defined, and why is it fundamental to AI models?

Probability measures how likely an event is to occur.

Its value always lies between 0 and 1.

  • 0 → impossible
  • 1 → certain

Formal view

  • Probability = (number of favorable outcomes) / (total possible outcomes), when outcomes are equally likely.

Example

Rolling a fair die:

  • Probability of getting 4 = 1 / 6

Why AI models rely on probability

AI systems work under uncertainty:

  • Data is incomplete
  • Inputs are noisy
  • Outputs are predictions, not guarantees

Examples in AI

  • Classification models output probabilities:
    • “Spam = 0.92, Not Spam = 0.08”
  • Language models predict the probability of the next word
  • Recommendation systems rank items by likelihood of user interest

Without probability, models could not express confidence or make ranked decisions.


What is the difference between independent and dependent events in ML pipelines?

Two events are independent if the occurrence of one does not affect the other.

They are dependent if one event changes the likelihood of the other.

Independent events example

  • User clicking an ad today
  • Coin toss result

Clicking the ad does not affect the coin toss outcome.

Dependent events example

  • User visits a product page
  • User adds the product to cart

Visiting the page increases the chance of adding to cart.

In ML pipelines

  • Independent features: word order ignored in Bag-of-Words
  • Dependent features: sequence of words in language models

Incorrect independence assumptions can lead to poor model performance.


How do you compute the joint probability of independent events?

For independent events, the joint probability is the product of individual probabilities.

Formula

  • P(A and B) = P(A) × P(B)

Example

  • P(user logs in today) = 0.6
  • P(user clicks notification) = 0.5

Joint probability:

  • P(login AND click) = 0.6 × 0.5 = 0.3

This rule is heavily used in models that assume independence, such as Naive Bayes.


How do mutually exclusive and independent events differ, and why does it matter in modeling?

Mutually exclusive events

  • Cannot occur at the same time

Example

  • User clicks “Buy Now”
  • User clicks “Cancel” on the same screen

Only one can happen.

Independent events

  • Occurrence of one does not affect the other

Key difference

  • Mutually exclusive events cannot be independent (unless probability is zero).

Why it matters

  • Treating mutually exclusive events as independent leads to incorrect probability calculations.
  • Many ML models rely on independence assumptions for simplicity.

Example

In multi-class classification:

  • A data point cannot belong to two classes at once (mutually exclusive).
  • Probabilities must sum to 1.

What is conditional probability, and how is it used in classification models?

Conditional probability is the probability of an event occurring given that another event has already occurred.

Notation

  • P(A | B): probability of A given B

Example

  • Probability an email is spam given it contains the word “free”

Classification models estimate:

  • P(class | features)

Example

  • P(spam | “free”, “win”, “offer”)

The model predicts the class with the highest conditional probability.


How is conditional probability mathematically represented?

Formula

  • P(A | B) = P(A and B) / P(B)

This means:

  • Among all cases where B happens, how often does A also happen?

Example

  • 100 emails contain the word “free”
  • 60 of those are spam

P(spam | free) = 60 / 100 = 0.6


Can conditional probability ever be greater than marginal probability? Explain with intuition.

Yes, it can.

Why

  • Knowing additional information can increase confidence.

Example

  • P(user buys product) = 0.1
  • P(user buys product | user added to cart) = 0.7

The condition (“added to cart”) provides strong evidence.

Intuition

  • Marginal probability is an overall average.
  • Conditional probability focuses on a specific context where the event is more likely.

How does conditional probability relate to feature dependencies in ML models?

Conditional probability captures how features influence outcomes together.

Example

  • P(fraud | high amount, new device)

    is more informative than:

  • P(fraud | high amount)

  • P(fraud | new device)

Models like:

  • Logistic Regression
  • Naive Bayes
  • Bayesian Networks

explicitly or implicitly learn these dependencies.

Ignoring dependencies can oversimplify the problem and reduce accuracy.


What is Bayes’ Theorem, and how would you explain it intuitively in an interview?

Bayes’ Theorem explains how to update beliefs when new evidence is observed.

Intuition

  • Start with an initial belief.
  • See new data.
  • Adjust the belief accordingly.

Example

  • Initially, very few emails are spam.
  • Seeing the word “lottery” increases the chance that an email is spam.

Bayes’ Theorem formalizes this update process.


What do prior, likelihood, and posterior represent in Bayesian models?

Prior

  • Initial belief before seeing data
  • Example: 5% of emails are spam

Likelihood

  • How likely the observed data is if the hypothesis is true
  • Example: spam emails often contain “win”

Posterior

  • Updated belief after seeing data
  • Example: probability that the email is spam after seeing the word “win”

Posterior = updated decision-making probability.


Why is Bayes’ Theorem critical in probabilistic ML and NLP tasks?

Bayes’ Theorem allows models to:

  • Combine prior knowledge with observed data
  • Handle uncertainty explicitly
  • Update predictions continuously

Examples

  • Naive Bayes text classification
  • Spam detection
  • Medical diagnosis
  • Speech recognition
  • Information retrieval

It enables reasoning under uncertainty rather than fixed rules.


How does Bayes’ Theorem help update beliefs when new data arrives?

Bayesian updating happens step by step.

Example

  • Prior: P(spam) = 0.05
  • Evidence: email contains “free”

After applying Bayes’ Theorem:

  • Posterior: P(spam | free) = higher than 0.05

If another word appears (“lottery”):

  • Posterior is updated again using the previous posterior as the new prior.

This continuous updating makes Bayesian models adaptable and data-driven.


What is a normal distribution, and why does it frequently appear in ML datasets?

A normal distribution is a symmetric, bell-shaped distribution where:

  • Most values are near the mean
  • Fewer values appear as you move away from the center
  • Mean = Median = Mode

Example

Heights of people in a city:

  • Most people are near average height
  • Very short or very tall people are rare

This pattern naturally forms a bell curve.

Why it appears in ML

  • Many features are the result of many small independent factors
  • When multiple small effects combine, values tend to cluster around an average

ML examples

  • Measurement noise
  • Sensor readings
  • Model errors
  • Aggregated user behavior after scaling

Why do many ML algorithms assume data is normally distributed?

Normal distribution simplifies math and makes models more stable.

Reasons

  • Closed-form solutions exist
  • Gradients behave smoothly
  • Statistical assumptions become valid

Examples

  • Linear Regression assumes normally distributed errors
  • PCA works best when data is centered and symmetric
  • Gaussian Naive Bayes explicitly models features as normal

When data is approximately normal:

  • Optimization converges faster
  • Predictions are more reliable

What is the 68–95–99.7 rule, and where is it practically useful in ML?

The 68–95–99.7 rule describes how data spreads in a normal distribution.

  • 68% of values lie within 1 standard deviation of the mean
  • 95% lie within 2 standard deviations
  • 99.7% lie within 3 standard deviations

Example

Feature: user response time

  • Mean = 200 ms
  • Std Dev = 20 ms

Then:

  • ~68% responses: 180–220 ms
  • ~95% responses: 160–240 ms

Practical ML uses

  • Outlier detection
  • Setting anomaly thresholds
  • Monitoring model drift
  • Feature sanity checks

What are the consequences if training data does not follow a normal distribution?

Non-normal data can cause:

  • Poor model assumptions
  • Unstable training
  • Skewed predictions

Examples

  • Right-skewed income data
  • Heavy-tailed click data

Effects

  • Mean becomes misleading
  • Gradient updates become noisy
  • Models overreact to extreme values

Common fixes

  • Log transformation
  • Robust scaling
  • Using distribution-free models (trees, boosting)

Normality helps, but it is not mandatory if handled correctly.


What is correlation, and how is it quantified in ML workflows?Untitled

Correlation measures how strongly two variables move together.

Range

  • +1 → strong positive relationship
  • 0 → no linear relationship
  • −1 → strong negative relationship

Example

  • Temperature ↑ → Ice cream sales ↑ (positive correlation)

In ML

  • Used during feature analysis
  • Helps detect redundancy
  • Often measured using Pearson correlation

What does a correlation coefficient close to zero imply for feature selection?

A correlation close to zero means:

  • No linear relationship

Important nuance

  • Non-linear relationships may still exist

Example

  • Feature: x
  • Target: x²

Correlation ≈ 0, but feature is still useful.

ML implication

  • Do not drop features only because correlation is low
  • Check non-linear models or transformations first

Why is correlation not sufficient to establish causation in AI systems?

Correlation shows association, not cause.

Reasons

  • Hidden variables may drive both
  • Reverse causation is possible
  • Coincidental patterns exist in large datasets

Example

  • Users who watch more ads buy more products

    This does not mean ads caused purchases.

Models that confuse correlation with causation:

  • Learn spurious patterns
  • Fail in real-world deployment

Can two features be highly correlated but have no causal relationship? Explain with an ML example.

Yes.

Example

  • Number of app downloads
  • Number of customer support tickets

Both increase as user base grows.

High correlation exists, but:

  • Downloads do not cause tickets
  • User growth causes both

If user base changes, the learned relationship breaks.


Why is sampling necessary when working with large-scale AI datasets?

Sampling allows:

  • Faster experimentation
  • Lower compute cost
  • Quicker feedback loops

Example

Training on:

  • 10 million rows → expensive
  • 200k representative rows → efficient

Sampling is used for:

  • Exploratory data analysis
  • Hyperparameter tuning
  • Model debugging

What common sampling biases can negatively impact AI models and lead to misleading conclusions?

Selection bias

  • Sample does not represent real population
  • Example: training only on active users

Survivorship bias

  • Ignoring failed or missing cases
  • Example: only successful transactions analyzed

Temporal bias

  • Old data does not reflect current behavior

Class imbalance bias

Impact

Linear Algebra for ML

What is a vector and why are they central to machine learning?

A vector is an ordered list of numbers — in ML, almost everything is a vector. A word embedding is a vector (e.g., [0.2, -0.5, 0.8, ...]). A row in a dataset is a vector of features. An image is a vector of pixel values (after flattening). A model's prediction is a vector of class probabilities.

Vectors live in n-dimensional space. Two vectors are "similar" if they point in the same direction. This geometric intuition is the foundation of semantic similarity in embeddings, nearest-neighbor search in RAG, and attention in transformers.

What is a matrix and what does matrix multiplication mean in neural networks?

A matrix is a 2D array of numbers (rows x columns). In a neural network, the weight matrix W transforms an input vector x into an output vector y via matrix multiplication: y = Wx. Each row of W represents a different linear combination of the input features — it's a learned projection from input space into output space.

In a transformer attention head: Q = X x W_Q, K = X x W_K — these are matrix multiplications that project input tokens into query and key spaces. Matrix multiplication is the core computation in every forward pass.

What is the dot product and why does it measure similarity?

The dot product of two vectors a and b is: a · b = a₁b₁ + a₂b₂ + ... + aₙbₙ. It equals |a||b|cos(θ), where θ is the angle between them.

  • When θ = 0 (vectors point the same direction), cos(θ) = 1 → maximum dot product
  • When θ = 90° (orthogonal), cos(θ) = 0 → dot product = 0

This is why dot product (and cosine similarity) measure semantic similarity: two embeddings for "happy" and "joyful" have a high dot product because they point in similar directions in embedding space. In attention: Attention(Q,K,V) = softmax(QKᵀ/√d)V — the QKᵀ term is a matrix of dot products between queries and keys.

What is a transpose and why do you see it everywhere in ML math?

The transpose of a matrix swaps its rows and columns: (A)ᵢⱼ = (Aᵀ)ⱼᵢ. In practice: if A is (m x n), then Aᵀ is (n x m).

You see it in: attention (QKᵀ = query matrix multiplied by transposed key matrix), loss computation (yᵀŷ inner product), and gradient derivations. Mental model: transpose "rotates" a matrix so that row vectors become column vectors.

What are eigenvalues and eigenvectors, and why do they matter for ML?

For a square matrix A, eigenvector v satisfies Av = λv, where λ is the eigenvalue. In plain English: eigenvectors are the "special directions" that a transformation only stretches or shrinks (by factor λ), never rotates.

Why it matters for ML:

  • PCA (Principal Component Analysis) finds the eigenvectors of the data covariance matrix — they define the directions of maximum variance
  • LoRA uses low-rank decomposition related to SVD (which generalises eigendecomposition) to approximate weight matrix updates efficiently
What is Singular Value Decomposition (SVD) and how does LoRA use it?

SVD decomposes any matrix M into M = UΣVᵀ, where U and V are orthogonal matrices and Σ is diagonal with singular values. The key insight: most matrices can be well approximated by keeping only the largest k singular values — a low-rank approximation.

LoRA (Low-Rank Adaptation) exploits this: instead of updating the full weight matrix ΔW (which could be 4096x4096 = 16M parameters), it decomposes it as ΔW = AB where A is (4096xr) and B is (rx4096) with rank r much less than 4096. Training only A and B is r/2048 the parameters. This works because the meaningful changes to a pretrained weight matrix tend to have low intrinsic rank.

What is cosine similarity and how does it differ from Euclidean distance?

Cosine similarity = (a · b) / (|a| × |b|). It measures the angle between vectors, not their distance.

Euclidean distance = √(Σ(aᵢ - bᵢ)²). It measures the straight-line distance between endpoints.

For embeddings: cosine similarity is preferred because two sentences with the same meaning but different lengths will have embeddings in the same direction but different magnitudes. Cosine similarity captures the "direction of meaning" regardless of magnitude.

Practical range: 1.0 = identical direction, 0 = orthogonal (unrelated), -1 = opposite. Most embedding models output L2-normalised vectors, making cosine similarity equal to dot product (faster to compute).

What is a tensor in the context of deep learning?

A tensor is a generalisation of scalars, vectors, and matrices to any number of dimensions:

  • A scalar is a 0D tensor
  • A vector is a 1D tensor
  • A matrix is a 2D tensor
  • A batch of images is a 4D tensor: (batch_size x channels x height x width)

In PyTorch and TensorFlow, all data and model weights are tensors. Operations like matrix multiplication, convolution, and attention are all tensor operations. Understanding tensor shapes is a core debugging skill: shape mismatches cause most deep learning errors. Always check .shape after every operation.

Calculus & Optimisation for ML

What is a gradient and why is it the engine of machine learning?

A gradient is the multi-variable generalisation of a derivative. For a function f(x₁, x₂, ..., xₙ), the gradient ∇f is a vector of partial derivatives: ∇f = (∂f/∂x₁, ∂f/∂x₂, ..., ∂f/∂xₙ). Each partial derivative measures: "how much does f change if I change xᵢ a little?"

The gradient always points in the direction of steepest increase of f. To minimize f (like a loss function), move in the opposite direction: x = x - α∇f. This is gradient descent — the universal learning algorithm for neural networks. With millions of parameters, computing the gradient efficiently is why backpropagation exists.

What is backpropagation and how does the chain rule make it work?

Backpropagation computes gradients of the loss with respect to every weight in a neural network. It uses the chain rule: d(f(g(x)))/dx = f'(g(x)) × g'(x).

In a neural network: loss → output layer → hidden layers → input. The chain rule lets us "backpropagate" the loss gradient through each layer: ∂L/∂W₁ = (∂L/∂output) × (∂output/∂hidden) × (∂hidden/∂W₁).

PyTorch's autograd does this automatically — every operation on a tensor records itself in a computational graph, then .backward() traverses the graph in reverse applying the chain rule at each node.

What is gradient descent, and what are its variants (SGD, Adam)?
  • Full-batch gradient descent: compute gradient over the entire dataset — slow for large datasets
  • Stochastic Gradient Descent (SGD): compute gradient on a single sample — noisy but fast
  • Mini-batch SGD: compute gradient on a batch of 32-512 samples — the standard
  • Adam (Adaptive Moment Estimation): adapts the learning rate per parameter using running averages of gradients (momentum) and squared gradients (variance normalisation). Adam is the default optimizer for most LLMs and transformers.

Key hyperparameters: learning rate (α), β₁ (momentum decay, default 0.9), β₂ (variance decay, default 0.999). AdamW = Adam with weight decay — used for most modern LLM training.

What is a learning rate and why is it so important?

Learning rate (α) controls the step size of each gradient update: W = W - α∇L.

  • Too large: overshoots the minimum — loss diverges or oscillates
  • Too small: extremely slow convergence — model barely learns

Modern practice: use a learning rate schedule. Warmup: start small (prevents early instability). Cosine decay: reduce gradually following a cosine curve. For LLM fine-tuning: typical α ≈ 1e-4 to 1e-5. Too-large learning rates during fine-tuning cause "catastrophic forgetting" — overwriting pretrained knowledge.

What is the vanishing gradient problem and how do modern architectures solve it?

In deep networks, gradients are multiplied together during backpropagation. If gradients are consistently less than 1.0, multiplying 50 of them together approaches 0 — early layers receive essentially no learning signal.

Solutions:

  • Residual connections (ResNet, Transformer): skip connections add the input directly to the output (y = F(x) + x), providing gradient highways that bypass layers. This is why transformers can be trained at arbitrary depth.
  • Layer normalisation: normalises activations to prevent them from growing or shrinking across layers
  • ReLU activations: don't saturate in the positive domain (unlike sigmoid/tanh), avoiding gradient saturation
What is cross-entropy loss and why is it used for language model training?

Cross-entropy loss = -Σ y_true × log(y_pred). For language models: y_true is a one-hot vector (the actual next token), y_pred is the model's probability distribution over all tokens. Cross-entropy = -log(probability assigned to the correct token).

  • If the model assigns 0.9 probability to the correct token: loss = -log(0.9) = 0.105 (low loss, good prediction)
  • If the model assigns 0.01 probability: loss = -log(0.01) = 4.6 (high loss, bad prediction)

Minimizing cross-entropy is equivalent to maximizing the log-likelihood of the training data — that's the information-theoretic foundation of next-token prediction.

Information Theory for AI

What is entropy in information theory and why does it matter for AI?

Entropy measures the average uncertainty (or information content) of a probability distribution: H(X) = -Σ p(x) log p(x).

  • High entropy = high uncertainty = more information needed
  • Low entropy = low uncertainty = predictable

Example: a fair coin (p=0.5) has maximum entropy = 1 bit. A weighted coin (p=0.99 heads) has near-zero entropy — almost no uncertainty. For LLMs: perplexity = exp(H) — a language model's perplexity on text measures how "surprised" it is. Lower perplexity = better model.

What is KL divergence and where is it used in ML?

KL divergence (Kullback-Leibler) measures how different one probability distribution P is from another Q: DKL(P||Q) = Σ P(x) log(P(x)/Q(x)). Not symmetric: DKL(P||Q) ≠ DKL(Q||P). Always ≥ 0; equals 0 when P = Q.

  • In RLHF: the PPO objective includes a KL penalty between the policy being trained and the reference (SFT) model: L = reward - β × DKL(policy || reference)
  • In variational autoencoders (VAEs): KL divergence regularises the learned latent distribution toward a standard normal
  • In knowledge distillation: minimise KL divergence between student and teacher output distributions
What is mutual information and how is it used for feature selection?

Mutual information I(X;Y) measures how much knowing X tells you about Y: I(X;Y) = H(X) - H(X|Y). H(X|Y) is the entropy of X given you know Y — if knowing Y reduces uncertainty about X, I(X;Y) > 0.

Used in feature selection: compute I(feature; label) for each feature — features with higher mutual information are more predictive of the label. More robust than correlation because it captures non-linear relationships.

What is perplexity and why is it the standard evaluation metric for language models?

Perplexity = exp(-1/N Σ log P(wᵢ|w₁...wᵢ₋₁)). In plain English: the geometric mean of the inverse probability the model assigns to each token. Lower is better.

  • A perplexity of 10 means the model is about as confident as choosing randomly among 10 equally likely options
  • A perplexity of 100 means the model finds the text very surprising

Limitation: doesn't directly measure human-perceived quality. Modern evaluations supplement perplexity with task-based benchmarks.

What is the connection between information theory and compression, and why does it matter for LLMs?

Shannon's source coding theorem: the minimum average code length to represent symbols from distribution P is H(P) bits. A perfect compressor is equivalent to a perfect probability model.

LLMs can be viewed as extremely powerful text compressors — by assigning high probability to natural language, they implicitly compress it efficiently. The compression rate of an LLM on a text corpus is a proxy for how well the LLM models that distribution. Researchers have used this framing to argue that large LLMs "understand" text in a meaningful sense: you cannot compress something well without understanding its structure.
