Machine Learning Basics & Terminology

What is Machine Learning, and how is it different from traditional programming?

Machine Learning is a way of making computers learn patterns from data instead of writing exact rules for every situation.

In traditional programming:

  • You write rules manually
  • You give input + rules
  • The computer produces output

Example (traditional programming):

  • Rule: If marks ≥ 40 → Pass, else Fail
  • Input: 65
  • Output: Pass

Here, the computer does not learn. It only follows rules written by a human.

In Machine Learning:

  • You give input data + output data
  • The computer figures out the rules on its own
  • These learned rules are used to predict future outputs

Example (Machine Learning):

  • Input data: marks of students
  • Output data: Pass/Fail result
  • The system studies many examples and learns how marks relate to results
  • Later, when you give new marks (say 52), it predicts Pass or Fail without any hardcoded rule

So the main difference:

  • Traditional programming → rules first, data later
  • Machine Learning → data first, rules are learned automatically

What are the main types of Machine Learning?

There are three commonly used types:

Supervised Learning

  • Data comes with correct answers (labels)
  • The model learns by comparing its prediction with the correct answer

Example:

  • Email text → Spam or Not Spam
  • House size → House price

Unsupervised Learning

  • Data has no answers or labels
  • The model tries to find patterns or groups on its own

Example:

  • Grouping customers based on shopping behavior
  • Finding similar movies based on viewing patterns

Reinforcement Learning

  • The model learns by trial and error
  • It gets rewards for correct actions and penalties for wrong ones

Example:

  • Teaching a game-playing bot to win a game
  • Training a robot to walk without falling

What is the difference between AI, ML, and Deep Learning?

Artificial Intelligence (AI)

  • A broad field
  • Goal: Make machines behave like humans
  • Includes rule-based systems, search algorithms, ML, etc.

Example:

  • A chess-playing program that follows predefined rules

Machine Learning (ML)

  • A subset of AI
  • Focuses on learning from data
  • Improves performance with experience

Example:

  • A recommendation system that improves as more users watch movies

Deep Learning

  • A subset of Machine Learning
  • Uses neural networks with many layers
  • Works very well with images, audio, and text

Example:

  • Face recognition in photos
  • Voice assistants understanding speech

Relation:

  • AI → ML → Deep Learning

What does a “model” mean in Machine Learning?

A model is the learned representation of patterns from data.

During training:

  • The model studies input and output data
  • It learns relationships between them
  • These learned relationships are stored as numbers (parameters)

Example:

  • You train a model with:
    • Input: house size
    • Output: house price
  • The model learns how size affects price
  • That learned logic is called the model

Later:

  • You give a new house size
  • The model predicts the price using what it learned earlier

So, a model is basically:

  • The brain of a Machine Learning system
  • Used to make predictions on new data

What is training data vs inference data?

Training Data

  • Data used to teach the model
  • Contains examples the model learns from
  • Usually includes inputs and correct outputs

Example:

  • Past exam scores and final grades
  • Old customer data and whether they bought a product

Inference Data

  • New, unseen data
  • Used after training
  • The model makes predictions on this data

Example:

  • A new student’s marks
  • A new customer visiting a website

In short:

  • Training data → learning phase
  • Inference data → prediction phase

What is supervised learning?

Supervised learning is a type of learning where the model is trained using data that already has correct answers.

Each data point contains:

  • Input
  • Correct output (label)

Example:

  • Input: email text
  • Output: Spam or Not Spam

The model:

  • Makes a prediction
  • Compares it with the correct answer
  • Adjusts itself to reduce mistakes

Over time, predictions become more accurate.

What is unsupervised learning?

Unsupervised learning works with data that has no labels or correct answers.

The model:

  • Does not know what the output should be
  • Looks for patterns, similarities, or groups

Example:

  • Customer data with age, spending, and location
  • No information about customer types
  • The model groups customers based on similarity

Common outcomes:

  • Clusters
  • Patterns
  • Relationships

What kind of problems are best solved using supervised learning?

Supervised learning is best when:

  • You know the correct output
  • You want accurate predictions

Examples:

  • Predicting house prices
  • Detecting spam emails
  • Predicting student pass or fail
  • Identifying diseases from medical reports

These problems already have historical data with answers.

What kind of problems require unsupervised learning?

Unsupervised learning is useful when:

  • No labeled data is available
  • You want to explore data

Examples:

  • Grouping customers for marketing
  • Finding hidden patterns in sales data
  • Organizing news articles by topic
  • Detecting unusual behavior in transactions

Here, the goal is understanding data, not predicting exact values.

How do labels change the learning process in ML?

Labels act like guidance for the model.

With labels:

  • The model knows what the correct answer is
  • It can measure how wrong it is
  • It improves by correcting mistakes

Example:

  • Image with label “Cat”
  • If the model predicts “Dog”, it knows it made a mistake

Without labels:

  • The model has no way to know right or wrong
  • It only groups or analyzes data based on similarity

So:

  • Labels turn learning into a goal-oriented process
  • Without labels, learning becomes exploratory

What is regression in Machine Learning?

Regression is used when the output you want to predict is a number.

The model learns the relationship between input values and a continuous numeric output.

Examples:

  • Input: house size → Output: house price
  • Input: number of hours studied → Output: exam marks
  • Input: temperature → Output: electricity consumption

Example explanation:

  • You collect data of house sizes (1000 sqft, 1200 sqft, 1500 sqft)
  • Along with their prices (50 lakhs, 60 lakhs, 75 lakhs)
  • The model learns how size affects price
  • When you give a new size (1300 sqft), it predicts a price

Here, the output can be any value, not limited to fixed categories.

What is classification in Machine Learning?

Classification is used when the output belongs to a fixed set of categories or classes.

The model learns how inputs map to labels.

Examples:

  • Email → Spam / Not Spam
  • Student marks → Pass / Fail
  • Image → Cat / Dog / Bird

Example explanation:

  • You give emails as input
  • Each email has a label: Spam or Not Spam
  • The model learns patterns like keywords, structure, sender
  • For a new email, it predicts one class

Here, the output is not a number, but a category.


[Image/Media]

How do regression and classification differ in output and loss functions?

Output difference

  • Regression → numeric value
    • Example: 72.5, 120000, 3.8
  • Classification → class label or probability
    • Example: Spam, Pass, Cat

Loss function difference

Loss function measures how wrong the model is.

Regression loss functions:

  • Mean Squared Error (MSE)
  • Mean Absolute Error (MAE)

Example:

  • Actual price: 50
  • Predicted price: 45
  • Error = difference between numbers

Classification loss functions:

  • Cross-Entropy Loss
  • Log Loss

Example:

  • Actual label: Cat
  • Predicted label: Dog
  • The loss penalizes wrong class prediction

So:

  • Regression measures numeric distance
  • Classification measures class correctness

Example dataset:

Regression version:

  • Output: exact marks (78)
  • Problem: Predict marks

Classification version:

  • Output: Pass / Fail
  • Rule: Marks ≥ 40 → Pass, else Fail
  • Problem: Predict result category

Another example:

  • Customer spending amount: 12,000

Regression:

  • Predict exact spending amount

Classification:

  • High spender / Low spender

Same data, different problem framing.

Regression use cases

  • Predicting house prices
  • Forecasting sales numbers
  • Predicting delivery time
  • Estimating temperature or rainfall

Classification use cases

  • Spam detection
  • Disease diagnosis (Yes / No)
  • Face recognition
  • Fraud detection (Fraud / Not Fraud)

Simple way to remember:

  • “How much?” → Regression

Why do we split data into training and testing sets?

The goal is to check whether the model works on new data, not just data it has already seen.

Training set:

  • Used to teach the model

Testing set:

  • Used to evaluate performance on unseen data

Example:

  • You memorize answers before an exam → looks good
  • You write a new exam → real test of understanding

If you test on training data:

  • The model may appear very accurate
  • But fail on real-world data

[Image/Media]

What is the purpose of a validation set?

The validation set is used to make decisions while training, not for final evaluation.

Used for:

  • Choosing model type
  • Tuning hyperparameters (learning rate, depth, epochs)

Example:

  • You train multiple models
  • Check performance on validation data
  • Pick the best model
  • Final performance is checked on test data

So:

  • Training → learning
  • Validation → decision making
  • Test → final evaluation

[Image/Media]

What happens if you tune hyperparameters on the test set?

The test set stops being “unseen”.

Problems caused:

  • Model indirectly learns test data patterns
  • Test accuracy becomes misleading
  • Real-world performance drops

Example:

  • You adjust your study strategy after seeing exam questions
  • Then claim high performance on the same exam

This gives false confidence.

Correct approach:

  • Never touch the test set until the very end

[Image/Media]

What is data leakage, and how does improper splitting cause it?

Data leakage happens when information from the test set leaks into training.

Common causes:

  • Splitting data after normalization
  • Using future data to predict past events
  • Duplicate records across splits

Example:

  • Predicting tomorrow’s sales
  • Training data accidentally includes tomorrow’s value
  • Model looks perfect but fails in real use

Improper splitting makes the model cheat without knowing it.


[Image/Media]

How do you split data when the dataset is very small?

When data is limited, splitting must be careful.

Common approaches:

  • Cross-validation
  • K-fold validation

Example:

  • Dataset of 100 records
  • Split into 5 parts
  • Train on 4 parts, test on 1 part
  • Repeat so each part becomes test once

Benefits:

  • Better use of limited data
  • More reliable performance estimate

Another option:

  • Use larger training set
  • Smaller test set
  • Avoid too many splits

Goal:

  • Maximize learning
  • Still test on unseen data

What is overfitting in Machine Learning?

Overfitting happens when a model learns the training data too well, including noise and random details that do not actually matter.

What the model does:

  • Memorizes training examples
  • Learns patterns that are specific only to training data
  • Fails to perform well on new data

Example:

  • You train a model to predict exam marks
  • Training accuracy: 99%
  • Validation accuracy: 60%

This means the model is not learning general rules, it is memorizing.

Real-life analogy:

  • A student memorizes answers to previous question papers
  • Performs well on the same questions
  • Fails when questions are slightly changed

Overfitting gives high training performance but poor real-world results.


[Image/Media]

What is underfitting?

Underfitting happens when a model is too simple to capture the actual pattern in the data.

What the model does:

  • Ignores important relationships
  • Makes many mistakes even on training data
  • Never learns properly

Example:

  • Predicting house price using only number of rooms
  • Ignoring size, location, age of house
  • Training accuracy: 50%
  • Validation accuracy: 48%

Real-life analogy:

  • A student studies only headings without understanding concepts
  • Performs poorly even on simple questions

Underfitting results in poor performance everywhere.


[Image/Media]

How can you identify overfitting from training and validation curves?

Training and validation curves show how error or accuracy changes over time.

Overfitting pattern:

  • Training error keeps decreasing
  • Validation error decreases initially
  • Validation error starts increasing after some point

Example:

  • Epoch 1: Train error = 40%, Val error = 42%
  • Epoch 10: Train error = 10%, Val error = 15%
  • Epoch 50: Train error = 2%, Val error = 30%

This gap indicates:

  • Model is improving on training data
  • But getting worse on unseen data

The growing gap between curves is a clear sign of overfitting.


[Image/Media]

Why are complex models more prone to overfitting?

Complex models have:

  • Many parameters
  • High flexibility
  • Ability to fit almost any data

Examples of complex models:

  • Deep neural networks
  • High-degree polynomial models
  • Decision trees with many branches

Why this causes overfitting:

  • Model starts fitting noise
  • Learns random fluctuations
  • Treats noise as important patterns

Example:

  • A polynomial curve passing through every data point
  • Fits training data perfectly
  • Fails to predict new points correctly

More flexibility means more risk of memorization.


[Image/Media]

What are common ways to reduce overfitting?

Use more data

  • More examples reduce memorization
  • Model learns general patterns

Simplify the model

  • Reduce depth, layers, or features
  • Fewer parameters → less noise learning

Regularization

  • Penalizes overly complex solutions
  • Keeps weights small

Example:

  • L1 or L2 regularization

Early stopping

  • Stop training when validation error starts increasing

Dropout (for neural networks)

  • Randomly ignores some neurons during training
  • Prevents dependency on specific paths

What is bias in Machine Learning models?

Bias refers to error caused by wrong assumptions made by the model.

High bias means:

  • Model is too simple
  • Cannot capture true patterns
  • Makes systematic mistakes

Example:

  • Using a straight line to fit curved data
  • No matter how much data you add, the line cannot fit well

Real-life analogy:

  • Assuming all students learn at the same pace
  • Ignoring individual differences

High bias leads to underfitting.


[Image/Media]

What is variance in Machine Learning models?

Variance refers to how much a model’s predictions change when the training data changes.

High variance means:

  • Model is very sensitive to training data
  • Small data changes cause big prediction changes

Example:

  • Train model on Dataset A → high accuracy
  • Train model on Dataset B → very different predictions

Real-life analogy:

  • A student who changes answers based on minor hints
  • Lacks consistency

High variance leads to overfitting.


[Image/Media]

How does bias differ from variance intuitively?

Bias:

  • Error from oversimplification
  • Model misses important patterns

Variance:

  • Error from overcomplication
  • Model reacts too much to small changes

Example:

  • Bias: Always predicting average house price
  • Variance: Predicting wildly different prices for similar houses

One ignores data details, the other overreacts to them.


[Image/Media]

What happens when a model has high bias and low variance?

High bias + low variance means:

  • Model is simple and stable
  • Makes similar predictions everywhere
  • Consistently wrong

Example:

  • Predicting all exam scores as 50
  • Predictions don’t change much
  • Always inaccurate

This is classic underfitting:

  • Poor training performance
  • Poor validation performance

[Image/Media]

How does the bias–variance tradeoff affect model selection?

Increasing model complexity:

  • Decreases bias
  • Increases variance

Decreasing complexity:

  • Increases bias
  • Decreases variance

Goal:

  • Balance both to minimize total error

Example:

  • Very simple model → misses patterns
  • Very complex model → memorizes data
  • Optimal model → learns meaningful patterns without memorizing noise

Model selection involves finding this balance using validation data.

What is feature scaling, and why is it important?

Feature scaling means converting numerical features to a similar range so that no single feature dominates the learning process.

Problem without scaling:

  • One feature has values between 0–100000
  • Another feature has values between 0–10
  • The model gives more importance to the large-range feature

Example:

  • Features:
    • House size = 1200
    • Number of bedrooms = 2
  • The size value is much larger
  • The model focuses more on size, even if bedrooms are equally important

With scaling:

  • Both features are brought to comparable ranges
  • The model learns fairly from all features

Scaling improves learning stability and prediction quality.


[Image/Media]

What is the difference between normalization and standardization?

Normalization

  • Scales values to a fixed range, usually 0 to 1

  • Formula:

    (value − min) / (max − min)

Example:

  • Marks: 50, 75, 100
  • Normalized: 0, 0.5, 1

Used when:

  • Feature values have known bounds
  • Algorithms expect limited ranges

Standardization

  • Centers data around 0
  • Scales using mean and standard deviation
  • Resulting values can be negative or positive

Example:

  • Mean = 70, Std = 10
  • Value 80 → (80 − 70) / 10 = 1

Used when:

  • Data follows roughly a normal distribution
  • Algorithms depend on distance or gradients

[Image/Media]

Which ML algorithms are sensitive to feature scaling?

Algorithms that rely on distance or gradient calculations are sensitive.

Examples:

  • Linear Regression (with gradient descent)
  • Logistic Regression
  • K-Nearest Neighbors
  • Support Vector Machines
  • Neural Networks
  • K-Means clustering

Example:

  • KNN calculates distance between points
  • Larger-scale features dominate distance
  • Wrong neighbors get selected

Tree-based models:

  • Decision Trees
  • Random Forests
  • XGBoost

These are not sensitive because they split based on conditions, not distances.


[Image/Media]

Why does gradient descent converge faster with scaled features?

Gradient descent updates parameters step by step to reduce error.

Without scaling:

  • Loss surface is stretched
  • Gradient steps zig-zag
  • Takes many steps to reach minimum

With scaling:

  • Loss surface becomes more balanced
  • Gradients point more directly to minimum
  • Fewer steps needed

Example:

  • Feature A range: 1–10
  • Feature B range: 1–10000
  • Gradient steps oscillate heavily
  • Scaling removes this imbalance

This results in faster and more stable convergence.


[Image/Media]

When should you not apply feature scaling?

Avoid scaling when:

  • Using tree-based models
  • Features are already in the same range
  • Feature values are categorical IDs

Example:

  • Customer ID = 10234
  • Scaling this creates meaningless numbers

Also:

  • Never fit scaling on full dataset
  • Fit only on training data, then apply to validation and test data

Raw data often:

  • Is not directly useful

Example:

  • Raw feature: Date = 2025-01-15

These new features help the model learn better patterns.

How does good feature engineering improve model performance?

Good features:

  • Highlight important patterns
  • Reduce noise
  • Make learning easier

Example:

  • Predicting house prices
  • Raw features: address text
  • Engineered features:
    • City
    • Distance to city center
    • Nearby schools count

Better features often improve results more than complex models.

What is the difference between raw features and engineered features?

Raw features

  • Directly collected
  • Often unprocessed

Example:

  • Timestamp
  • Text description
  • Full address

Engineered features

  • Derived or transformed
  • More informative

Example:

  • Hour of day from timestamp

Engineered features are easier for models to understand.

How do you handle categorical features in ML models?

Categorical features represent categories, not numbers.

Examples:

  • Gender: Male / Female
  • City: Delhi, Mumbai, Chennai

Common techniques:

Label Encoding

  • Assigns numbers to categories
  • Example: Male = 0, Female = 1
  • Useful for tree models

One-Hot Encoding

  • Creates separate columns for each category
  • Example:
    • City_Delhi
    • City_Mumbai
    • City_Chennai

Target Encoding

  • Replaces category with average target value
  • Used carefully to avoid leakage

What are common feature engineering mistakes in real-world ML projects?

Common mistakes:

  • Creating features using future information
  • Encoding categories before splitting data
  • Overengineering too many features
  • Ignoring domain knowledge
  • Scaling test data separately

Example mistake:

  • Using total monthly sales to predict mid-month demand

This leaks future information and inflates performance.

Why is model evaluation critical in Machine Learning?

Model evaluation checks whether the model actually works on unseen data.

Without evaluation:

  • Model may appear accurate
  • But fail in real-world use

Example:

  • Training accuracy: 98%
  • Test accuracy: 60%

Evaluation reveals real performance and reliability.

Why is accuracy not always a good evaluation metric?

Accuracy fails when classes are imbalanced.

Example:

  • 95 normal transactions
  • 5 fraudulent transactions

Model predicts “normal” always:

  • Accuracy = 95%
  • Fraud detection fails completely

So high accuracy does not always mean good performance.


[Image/Media]

What factors decide which evaluation metric to use?

Depends on:

  • Problem type (classification or regression)
  • Business cost of errors
  • Class balance

Examples:

  • Fraud detection → Precision, Recall
  • Medical diagnosis → Recall
  • House price prediction → MAE, RMSE

Metric choice depends on what errors matter most.


[Image/Media]

How does class imbalance affect model evaluation?

Class imbalance causes:

  • Biased accuracy
  • Model ignoring minority class

Example:

  • Disease cases: 1%
  • Healthy cases: 99%

Accuracy hides poor disease detection.

Metrics like:

  • Precision
  • Recall
  • F1-score
  • ROC-AUC

Give a clearer picture.


[Image/Media]

What is the difference between training performance and generalization performance?

Training performance:

  • How well the model fits training data

Generalization performance:

  • How well the model works on unseen data

Example:

  • Training accuracy: 99%
  • Validation accuracy: 70%

This gap indicates overfitting.

Good models:

  • Perform well on both
  • Have small performance gaps

What are true positives, false positives, true negatives, and false negatives?

Using fraud detection as example:

True Positive (TP)

  • Actual Fraud
  • Predicted Fraud
  • Example: Fraud transaction correctly detected

False Positive (FP)

  • Actual Normal
  • Predicted Fraud
  • Example: Normal transaction wrongly flagged as fraud

True Negative (TN)

  • Actual Normal
  • Predicted Normal
  • Example: Normal transaction correctly allowed

False Negative (FN)

  • Actual Fraud
  • Predicted Normal
  • Example: Fraud missed by the system

These four values form the foundation of all classification metrics.


[Image/Media]

What is precision, and when is it more important than accuracy?

Precision measures how many predicted positives are actually correct.

Formula:

  • Precision = TP / (TP + FP)

Example:

  • Predicted Fraud = 30
  • Actual Fraud among them = 20
  • Precision = 20 / 30 = 0.66

Precision is important when false positives are costly.

Examples:

  • Spam filter wrongly blocking important emails
  • Fraud system blocking genuine customer payments
  • Content moderation wrongly removing valid posts

In such cases, accuracy can be misleading, but precision tells how trustworthy positive predictions are.


[Image/Media]

What is recall, and when is it more important than precision?

Recall measures how many actual positives were correctly identified.

Formula:

  • Recall = TP / (TP + FN)

Example:

  • Actual Fraud cases = 25
  • Detected Fraud = 20
  • Recall = 20 / 25 = 0.8

Recall is important when missing a positive case is dangerous.

Examples:

  • Disease detection
  • Fraud detection
  • Fire or security alerts

Here, it is better to raise some false alarms than miss real problems.


[Image/Media]

What does F1-score represent, and why is it useful?

F1-score combines precision and recall into a single value.

Formula:

  • F1 = 2 × (Precision × Recall) / (Precision + Recall)

What it captures:

  • Penalizes extreme imbalance
  • Requires both precision and recall to be reasonably high

Example:

  • Precision = 0.9
  • Recall = 0.4
  • F1-score becomes low because recall is poor

F1-score is useful when:

  • Classes are imbalanced
  • Both false positives and false negatives matter

[Image/Media]

What is a classification threshold?

A classification threshold is the probability cutoff used to decide the predicted class.

Example:

  • Model outputs probability = 0.72
  • Threshold = 0.5
  • Prediction = Positive

If threshold = 0.8:

  • Same probability → Negative

The threshold controls how strict or relaxed predictions are.


[Image/Media]

How does changing the threshold affect precision and recall?

Lower threshold:

  • More positives predicted
  • Recall increases
  • Precision usually decreases

Higher threshold:

  • Fewer positives predicted
  • Precision increases
  • Recall usually decreases

Example:

  • Threshold = 0.3 → many fraud alerts
  • Threshold = 0.9 → very few fraud alerts

Choosing threshold depends on business risk.


[Image/Media]

Why is ROC–AUC considered a threshold-independent metric?

ROC–AUC evaluates model performance across all possible thresholds.

Instead of fixing one threshold:

  • It checks how well the model separates classes overall
  • Measures ranking quality, not final labels

AUC value:

  • 0.5 → random guessing
  • 1.0 → perfect separation

So ROC–AUC shows model strength regardless of threshold choice.


[Image/Media]

What does the ROC curve represent intuitively?

ROC curve plots:

  • True Positive Rate (Recall)
  • False Positive Rate

As threshold changes:

  • One point moves along the curve
  • Shows tradeoff between catching positives and raising false alarms

Intuition:

  • Higher curve = better separation
  • Curve near diagonal = weak model

[Image/Media]

When would you prefer ROC–AUC over accuracy or F1-score?

Use ROC–AUC when:

  • Class distribution is imbalanced
  • You want to compare models fairly
  • Threshold is not yet finalized

Examples:

  • Credit risk scoring
  • Medical diagnosis models
  • Early-stage model comparison

ROC–AUC focuses on ranking quality, not decision cutoffs.


[Image/Media]

What is regularization in Machine Learning?

Regularization is a technique that discourages overly complex models.

It works by:

  • Adding a penalty for large weights
  • Preventing the model from fitting noise

Example:

  • Model fits every data point perfectly
  • Regularization pushes it to prefer simpler patterns

This improves performance on unseen data.


[Image/Media]

How does L2 regularization help prevent overfitting?

L2 regularization adds a penalty based on squared weights.

Effect:

  • Large weights become expensive
  • Model spreads importance across features
  • No single feature dominates

Example:

  • One feature weight = 100
  • L2 penalty pushes it down
  • Encourages balanced learning

L2 makes the model smoother and more stable.

What is L1 regularization, and how is it different from L2?

L1 regularization adds a penalty based on absolute weights.

Effect:

  • Pushes some weights exactly to zero
  • Removes unnecessary features

Difference:

  • L1 → feature selection
  • L2 → weight shrinking

Example:

  • Dataset with many useless features
  • L1 removes them automatically

How does regularization affect model complexity?

Stronger regularization:

  • Smaller weights
  • Simpler model
  • Less overfitting

Weaker regularization:

  • More flexible model
  • Higher risk of overfitting

Regularization controls the balance between learning patterns and ignoring noise.


[Image/Media]

When might regularization hurt model performance?

Regularization can hurt when:

  • Model is already too simple
  • Data is very small
  • Important features get overly penalized

Example:

  • Linear model with few features
  • Strong regularization forces weights near zero
  • Model underfits

So regularization must be tuned carefully using validation data.

What is the difference between model parameters and hyperparameters?

Model parameters are values the model learns automatically from data during training.

Examples:

  • Weights in linear regression
  • Weights and biases in neural networks
  • Split thresholds in decision trees

These are adjusted by the learning algorithm itself.

Hyperparameters are values set before training starts and control how the learning process happens.

Examples:

  • Learning rate
  • Number of layers
  • Tree depth
  • Regularization strength
  • Number of neighbors in KNN

Example:

  • In house price prediction:
    • Parameters = learned relationship between size and price
    • Hyperparameters = how fast the model learns, how complex it can be

[Image/Media]

Why is hyperparameter tuning necessary?

Hyperparameters strongly influence:

  • Learning speed
  • Model complexity
  • Overfitting and underfitting

Bad choices:

  • Learning rate too high → unstable training
  • Learning rate too low → very slow learning
  • Tree depth too large → overfitting
  • Tree depth too small → underfitting

Example:

  • Two models with same data and algorithm
  • Different hyperparameters
  • Very different performance

Tuning helps find a balance that generalizes well.

What is grid search, and how does it work?

Grid search tries all possible combinations of given hyperparameter values.

Example:

  • Learning rate: [0.01, 0.1]
  • Depth: [3, 5]

Grid search tries:

  • (0.01, 3)
  • (0.01, 5)
  • (0.1, 3)
  • (0.1, 5)

Each combination:

  • Trains a model
  • Evaluates using validation data
  • Best combination is selected

Drawback:

  • Expensive
  • Grows rapidly with more parameters

What is random search, and why is it often preferred over grid search?

Random search samples random combinations instead of trying everything.

Example:

  • Learning rate: any value between 0.001 and 1
  • Depth: any value between 2 and 20
  • Randomly pick 20 combinations

Why it works better:

  • Explores wider space
  • Finds good values faster
  • Avoids wasting time on unimportant parameters

In practice:

  • Few hyperparameters matter a lot
  • Random search focuses more on them

  • Give misleading performance
  • Increase training cost

Example:

  • Trying hundreds of combinations
  • Selecting model that works best only on validation set
  • Fails on real-world data

Also:

  • High compute usage
  • Slower development
  • Harder reproducibility

Tuning should be controlled and validated carefully.

The Bitter Lesson & Data-Centric AI

What is Rich Sutton's "bitter lesson" and what does it predict about AI research?

Rich Sutton's 2019 essay "The Bitter Lesson" argued that 70 years of AI research show one recurring pattern: methods that leverage computation at scale consistently outperform methods that encode human knowledge. Hand-designed features, expert systems, and domain-specific heuristics have repeatedly been surpassed by simple, general methods run on more compute.

Examples: chess (deep search > expert evaluation functions), speech recognition (deep learning > phoneme models), computer vision (CNNs > hand-crafted features), NLP (language models > linguistics-informed models).

The "bitter" part: ML researchers repeatedly invest years in clever domain-specific solutions that get swept away when compute scales up a general approach. The lesson for practitioners: don't over-invest in elaborate feature engineering or model-specific tricks when more data and compute may be the better path. The success of LLMs is the latest — and largest — vindication of this lesson.

What is data-centric AI and how does it differ from model-centric AI?

Model-centric AI: hold the dataset fixed, improve the model architecture and hyperparameters. Model-centric thinking dominated 2012-2020: most practitioners optimized architectures (ResNet vs. VGG, LSTM vs. GRU) while using standard benchmark datasets.

Data-centric AI: hold the model fixed, improve the data — its quality, labeling consistency, coverage of edge cases, and representation of the problem. Andrew Ng popularised this framing around 2021, noting that in many production systems, the dataset is the bottleneck, not the model.

Key data-centric practices:

  • Systematic error analysis (find where the model fails, collect targeted data)
  • Label quality auditing (inconsistent labels hurt more than insufficient labels)
  • Data augmentation for underrepresented classes

For LLMs: the quality of SFT and RLHF data often matters more than the model size — LIMA showed 1,000 carefully selected instruction examples matched models trained on more than 50,000 examples.

What is the scaling hypothesis and how does it explain the success of large language models?

The scaling hypothesis: intelligence emerges from training sufficiently large models on sufficiently large datasets with sufficient compute. Formalised by Kaplan et al. (2020) and Hoffmann et al. (2022). The core claim: there is no fundamental barrier to scaling language models to human-level and beyond — just compute and data.

Evidence for: GPT-2 could barely write coherent paragraphs (2019); GPT-4 solves complex math, writes code, and passes professional exams (2023). Same architecture, 1,000x more compute.

Evidence against (or complicating): some capabilities (complex reasoning, factual precision) haven't scaled as cleanly as early scaling laws predicted, motivating test-time compute as a complement to pretraining scale.

What it means for practitioners: incremental architecture improvements are less reliable than scaling; foundational capabilities of commercial models will continue to improve; the question is whether there are qualitative limits that compute alone cannot overcome.

Hands-On Code: Text Classification with ML & LLMs (Chapter 4 — Hands-On LLMs Book)

How do you build a text classifier using sentence embeddings + logistic regression — and why does this beat bag-of-words?

This is the "embedding + classifier" pattern that bridges classical ML and modern NLP. Sentence embeddings capture semantics — "great film" and "excellent movie" are close in embedding space, even though they share no words. Logistic regression on top of them gives you a lightweight, interpretable classifier.

from datasets import load_dataset
from sentence_transformers import SentenceTransformer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import classification_report
import numpy as np

# 1. Load dataset
dataset = load_dataset("rotten_tomatoes")
X_train_text = dataset["train"]["text"]
y_train      = dataset["train"]["label"]
X_test_text  = dataset["test"]["text"]
y_test       = dataset["test"]["label"]

# 2. Embed (semantic representation)
model      = SentenceTransformer("sentence-transformers/all-mpnet-base-v2")
X_train    = model.encode(X_train_text, batch_size=64, show_progress_bar=True)
X_test_emb = model.encode(X_test_text,  batch_size=64, show_progress_bar=True)

print(f"Train embedding shape: {X_train.shape}")   # (8530, 768)

# 3. Train logistic regression (fast — embeddings are fixed)
clf = LogisticRegression(max_iter=1000)
clf.fit(X_train, y_train)

# 4. Evaluate
y_pred = clf.predict(X_test_emb)
print(classification_report(y_test, y_pred, target_names=["negative", "positive"]))

Expected accuracy: ~87-89% on rotten_tomatoes. For comparison, a bag-of-words + logistic regression baseline gets ~79-82%. The embedding captures that "disappointing" and "underwhelming" both predict negative sentiment, even though they share no characters.

How do you do zero-shot classification with embeddings — no training data required?

Zero-shot classification exploits the fact that label names themselves can be embedded. If a document embedding is closest to the "positive" label embedding, we classify it as positive — no training data needed. This works remarkably well for clear label semantics.

from sentence_transformers import SentenceTransformer
from sklearn.metrics.pairwise import cosine_similarity
import numpy as np

model = SentenceTransformer("sentence-transformers/all-mpnet-base-v2")

# Label embeddings — the model already "knows" what these labels mean
labels       = ["positive review", "negative review"]
label_embeds = model.encode(labels)

# Test sentences
test_sentences = [
    "An absolute masterpiece. Best film I've seen this year.",
    "Boring and predictable. Walked out after 30 minutes.",
    "Some good moments but ultimately disappointing.",
    "Stunning visuals and a compelling story. Highly recommend!",
]

for sentence in test_sentences:
    sent_embed = model.encode([sentence])
    similarities = cosine_similarity(sent_embed, label_embeds)[0]
    pred = labels[np.argmax(similarities)]
    print(f"  [{pred.split()[0].upper():8s}]  sim=({similarities[0]:.2f}, {similarities[1]:.2f})  '{sentence[:50]}'")

This pattern is powerful for prototyping classifiers before you collect labeled data. The limitation: it works well when the label names clearly describe the content (positive/negative, sports/politics) but poorly when labels are ambiguous or domain-specific jargon.
