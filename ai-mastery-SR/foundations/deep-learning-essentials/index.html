Deep Learning vs Machine Learning

Why do we need Deep Learning when Machine Learning already exists?

Machine learning is a framework that uses past data to identify relationships among features. Machines can then predict new data using mathematical relationships by generating dynamic, accurate, and stable models.

On the other hand, deep learning is a segment of machine learning that uses complex algorithms based on biological neural networks to mimic the human brain and make decisions and actions like humans.

The brain’s functioning and structure inspire Deep Learning to train neural networks. Deep learning also performs various complex operations to extract hidden features and patterns.

The critical aspect differentiating the two is that deep learning can extract hidden features and patterns in the data. In contrast, other machine learning can reduce dimensions but not identify new features.

What are deep and shallow networks? Which one is better and why?

What is a Neural Network?

Before understanding deep vs shallow networks, let’s quickly recall:

A neural network is a model made of:

  • Input layer

  • Hidden layer(s)

  • Output layer

The hidden layers are where learning happens.

The number of hidden layers decides whether the network is shallow or deep.


What is a Shallow Neural Network?

A shallow network has:

  • Only one hidden layer

  • OR very few hidden layers (usually 1)

Structure:

Input → Hidden Layer → Output

Example:

  • Logistic Regression (no hidden layer)

  • Neural network with 1 hidden layer


? Beginner Analogy

Think of a shallow network like:

A student who solves a math problem in one step.

It can solve simple problems, but struggles with complex reasoning.


? Characteristics of Shallow Networks

  • Simple architecture

  • Fewer parameters

  • Easier to train

  • Less computational power needed

  • Limited ability to learn complex patterns


3️⃣ What is a Deep Neural Network?

A deep network has:

  • Multiple hidden layers (2, 5, 10, 50… even 100+)

Structure:

Input → Hidden → Hidden → Hidden → Output

Each hidden layer learns something more complex than the previous one.


? Beginner Analogy

Think of a deep network like:

A team of specialists solving a problem step by step.

Example in image recognition:

Layer 1 → Detect edges
Layer 2 → Detect shapes
Layer 3 → Detect object parts
Layer 4 → Recognize full object

That’s how deep learning works.


? Characteristics of Deep Networks

  • Many hidden layers

  • Large number of parameters

  • Can learn complex patterns

  • Requires more data

  • Requires more computational power (GPU)

What is model capacity?

Model capacity is the ability of a machine learning model to learn and represent complex patterns from data.

In simple terms, it describes how flexible or powerful a model is.

If a model has low capacity, it can only learn simple relationships.
If a model has high capacity, it can learn very complex relationships.

What determines model capacity:

  1. Number of parameters
    More weights and biases increase capacity. A neural network with millions of parameters has higher capacity than one with a few thousand.

  2. Number of layers
    Deeper networks (more hidden layers) usually have higher capacity because they can build hierarchical representations.

  3. Number of neurons per layer
    Wider layers increase the number of learnable parameters and therefore increase capacity.

  4. Model type
    Linear regression has low capacity because it can only model linear relationships.
    Polynomial regression has higher capacity.
    Deep neural networks and large language models have very high capacity.

Example:

If the true relationship in data is a straight line like:

y = 2x + 3

A simple linear model can learn it easily.

But if the relationship is:

y = x³ + sin(x)

A linear model cannot represent this complexity. It does not have enough capacity.

Low capacity model:

  • Cannot capture the true pattern.

  • Performs poorly even on training data.

  • This situation is called underfitting.

  • It has high bias and low variance.

High capacity model:

  • Can fit very complex patterns.

  • May even memorize the training data.

  • Performs very well on training data but poorly on unseen data.

  • This is called overfitting.

  • It has low bias and high variance.

Connection to bias-variance tradeoff:

Low capacity → High bias, Low variance
High capacity → Low bias, High variance

The goal is to choose a model with enough capacity to capture the true pattern, but not so much that it memorizes noise.

Interview-ready answer:

Model capacity refers to the ability of a model to fit a wide range of functions or patterns. It depends on factors like the number of parameters, layers, and neurons. A low-capacity model may underfit, while a high-capacity model may overfit. The key is to balance capacity so the model generalizes well to unseen data.

What is Convex Hull? Where is it used?

Convex Hull is a concept from computational geometry.

Definition:

The convex hull of a set of points is the smallest convex shape that contains all the points.

Another way to say it:

If you have several points on a plane, the convex hull is the tightest convex boundary that encloses all of them.

Simple intuition:

Imagine you put nails on a board at different positions. Now stretch a rubber band around all the nails and release it. The rubber band will snap tightly around the outermost nails. The shape formed by the rubber band is the convex hull.

Important properties:

  1. It is always a convex polygon (in 2D).

  2. All given points lie either inside it or on its boundary.

  3. The boundary is formed only by the outermost points.

What does convex mean?

A shape is convex if for any two points inside the shape, the straight line connecting them lies entirely inside the shape.

Examples:

If you have points forming a circle-like distribution, the convex hull will look like a polygon approximating the circle.

If some points are inside the cluster and some are on the edge, only the edge points form the convex hull. Interior points are ignored when forming the boundary.

Mathematical perspective:

Given a set of points S, the convex hull is the smallest convex set that contains S.

In higher dimensions:

  • In 2D → convex hull is a polygon.

  • In 3D → convex hull is a polyhedron.

  • In higher dimensions → convex polytope.

Common algorithms to compute convex hull:

  • Graham Scan

  • Jarvis March (Gift Wrapping)

  • QuickHull

  • Monotone Chain algorithm

Time complexity (2D case):

Most efficient algorithms run in O(n log n), where n is the number of points.

Where is Convex Hull used?

  1. Computer Graphics

  • Collision detection

  • Shape boundary detection

  • Object wrapping

  • Simplifying complex shapes

For example, in a game engine, to check whether two objects might collide, their convex hulls are often used as approximations.

  1. Machine Learning

  • In Support Vector Machines (SVM)

For linearly separable classes, the optimal separating hyperplane depends on the convex hulls of the two classes. If the convex hulls of two classes do not intersect, they are linearly separable.

  • Outlier detection

Points far from the convex hull boundary can sometimes indicate anomalies or extreme points.

  1. Pattern Recognition and Image Processing

  • Boundary detection of objects in images

  • Shape analysis

  • Finding the outer contour of a cluster of pixels

  1. Geographic Information Systems (GIS)

  • Finding the boundary of a set of geographic locations

  • Defining service areas or coverage regions

  1. Robotics and Path Planning

  • Determining safe regions

  • Obstacle boundary representation

  1. Computational Geometry and Optimization

  • Used in solving linear programming problems

  • Many geometric algorithms use convex hull as a subroutine

What is an artificial neuron, and how does it relate to a biological neuron?

An artificial neuron is a mathematical function that:

  • Takes multiple inputs
  • Applies weights
  • Adds a bias
  • Passes the result through an activation function

Biological analogy

  • Dendrites → inputs
  • Synapse strength → weights
  • Cell firing threshold → bias
  • Axon firing → output

Example

Inputs: [x1, x2, x3]

Weights: [w1, w2, w3]

Output = activation(w1·x1 + w2·x2 + w3·x3 + bias)

The neuron “fires” if the weighted sum is strong enough.


What are weights and biases, and why are they critical in neural networks?

Weights

  • Control how important each input is
  • Learned during training

Bias

  • Shifts the activation threshold
  • Allows flexibility even when inputs are zero

Example

Predicting house price:

  • Weight for “area” might be high
  • Weight for “number of doors” might be low
  • Bias sets the base price

Without weights → all inputs treated equally

Without bias → model becomes too rigid


What is the role of layers in a deep neural network?

Layers organize neurons into levels of abstraction.

Types

  • Input layer: receives raw data
  • Hidden layers: extract patterns
  • Output layer: produces prediction

Example (image)

  • Layer 1: detects edges
  • Layer 2: detects shapes
  • Layer 3: detects objects

Each layer builds on the previous one.


How does increasing the number of layers change model capacity and risk?

More layers:

  • Increase representational power
  • Allow learning complex functions

But also:

  • Increase risk of overfitting
  • Increase training difficulty
  • Increase compute cost

Example

  • 1–2 layers → simple patterns
  • 10+ layers → very complex patterns

Deep models require:

  • More data
  • Regularization
  • Careful architecture design

Why are deep neural networks more prone to overfitting?

Deep networks:

  • Have very high capacity
  • Can memorize training data

Example

Model learns:

  • Exact images
  • Exact noise patterns

Result:

  • High training accuracy
  • Poor test accuracy

What is dropout, and how does it reduce overfitting?

Dropout randomly turns off neurons during training.

Effect

  • Network cannot rely on specific neurons
  • Forces redundancy
  • Encourages robust features

Example

During one pass:

  • Some neurons are inactive

    Next pass:

  • Different neurons are inactive

At inference:

  • All neurons are used, but scaled

How does L1/L2 regularization help in deep learning models?

L1 regularization

  • Encourages sparsity
  • Pushes some weights to zero

L2 regularization

  • Penalizes large weights
  • Keeps weights small and smooth

Effect

  • Reduces model complexity
  • Improves generalization
  • Prevents extreme weight values

Why are Convolutional Neural Networks better suited for image data than fully connected networks?

Images have:

  • Spatial structure
  • Local patterns

CNNs:

  • Preserve spatial relationships
  • Use shared weights
  • Require fewer parameters

Example

Edge detected in top-left is useful in bottom-right too.


What problem does convolution solve in computer vision tasks?

Convolution:

  • Detects local features
  • Reduces parameters
  • Enables translation invariance

Instead of learning every pixel relationship separately, it learns reusable filters.

It is a generalization of:

  • Scalar → 0D tensor

  • Vector → 1D tensor

  • Matrix → 2D tensor

  • Higher dimensional array → 3D, 4D, or nD tensor

Examples:

Scalar (0D):
5

Vector (1D):
[1, 2, 3]

Matrix (2D):
[[1, 2],
[3, 4]]

3D Tensor:
Used for color images.
Height × Width × Channels

4D Tensor:

Batch Size × Height × Width × Channels

What is a computational graph? How is it useful for Deep Learning?

A computational graph is a directed graph that represents mathematical operations as nodes and data as edges.

In simple terms:

  • Nodes represent operations (addition, multiplication, activation, etc.).

  • Edges represent data (tensors) flowing between operations.

Example:

If we compute:

z = (x + y) * w

The computational graph would look like:

x ----
→ (+) → (*) → z
y ---- ↑
w

Here:

  • Addition is one node.

  • Multiplication is another node.

  • x, y, and w are inputs.

  • z is output.

Why computational graphs are useful:

  1. Automatic Differentiation
    Deep learning relies on backpropagation.
    Computational graphs make it easy to compute gradients automatically using the chain rule.

  2. Efficient Execution
    Frameworks optimize the graph to:

    • Reduce redundant computations

    • Use parallel processing

    • Run efficiently on GPUs

  3. Visualization
    Tools like TensorBoard allow visualization of the model architecture using computational graphs.

  4. Debugging
    Helps understand how data flows through the network.

There are two types:

Static computational graph

  • Defined once and then executed.

  • Used in older TensorFlow versions.

Dynamic computational graph

  • Built during runtime.

  • Used in PyTorch.

  • Easier to debug and more flexible.

Computational graphs are fundamental because they enable automatic gradient computation, which makes training deep neural networks possible.

What are Autoencoders? What is the use of Autoencoders?

An autoencoder is a type of neural network used to learn compressed representations of data.

It consists of two main parts:

  1. Encoder
    Compresses the input into a lower-dimensional representation called the latent vector.

  2. Decoder
    Reconstructs the original input from the latent representation.

Structure:

Input → Encoder → Latent Representation → Decoder → Output

The goal is to minimize reconstruction error between input and output.

Uses of autoencoders:

  • Dimensionality reduction (alternative to PCA)

  • Feature learning

  • Noise removal (Denoising Autoencoders)

  • Anomaly detection (high reconstruction error indicates abnormal data)

  • Data compression

  • Pretraining deep networks (historically)

Autoencoders are unsupervised because they do not require labeled data. The input itself acts as the target.

What are GANs?

GAN stands for Generative Adversarial Network.

It consists of two neural networks competing against each other:

  1. Generator
    Tries to generate realistic fake data.

  2. Discriminator
    Tries to distinguish between real data and fake data.

Training process:

  • The generator creates fake samples.

  • The discriminator evaluates whether they are real or fake.

  • The generator improves to fool the discriminator.

  • The discriminator improves to detect fake samples.

This adversarial process continues until the generator produces highly realistic outputs.

Uses of GANs:

  • Image generation

  • Deepfake creation

  • Super-resolution

  • Style transfer

  • Data augmentation

  • Text-to-image generation

GANs are powerful generative models capable of producing realistic synthetic data.

Explain the Boltzmann machine. What is a Restricted Boltzmann machine?

Boltzmann Machine is a type of stochastic recurrent neural network.

It is an energy-based model that learns probability distributions over inputs.

Characteristics:

  • Fully connected network

  • Connections exist between all visible and hidden units

  • Undirected connections

  • Uses probabilistic sampling

Training is computationally expensive because of the full connectivity between units.

Restricted Boltzmann Machine (RBM)

RBM is a simplified version of Boltzmann Machine.

Key restriction:

  • No connections between visible units

  • No connections between hidden units

  • Only connections between visible and hidden layers

Structure:

Visible Layer ↔ Hidden Layer

Because of this restriction:

  • Training becomes much more efficient.

  • Easier to compute gradients.

Uses of RBM:

  • Dimensionality reduction

  • Feature extraction

  • Collaborative filtering (used in recommendation systems)

  • Pretraining deep belief networks

RBMs were historically important in early deep learning research

What are Seq2Seq (encoder-decoder) models? How is it different from Autoencoders?

Seq2Seq (Sequence-to-Sequence) models are neural network architectures used to transform one sequence into another sequence.

Common in:

  • Machine translation

  • Text summarization

  • Speech recognition

  • Chatbots

Structure:

  1. Encoder
    Takes input sequence and converts it into a context vector.

  2. Decoder
    Takes context vector and generates output sequence.

Input Sequence → Encoder → Context Vector → Decoder → Output Sequence

Example:

Input: "Hello"
Output: "Bonjour"

Difference between Seq2Seq and Autoencoder:

Autoencoder:

  • Input and output are the same.

  • Goal is reconstruction.

  • Used for representation learning.

Seq2Seq:

  • Input and output are different sequences.

  • Goal is transformation.

  • Used for translation, summarization, etc.

Another difference:

Autoencoders typically compress information into a latent space for reconstruction.

Seq2Seq models use the encoded representation to generate a different output sequence.

Modern Seq2Seq models use attention mechanisms and transformers instead of basic RNNs.

What are ensemble learning methods?

Ensemble learning methods combine multiple models to improve overall performance.

The idea:

Multiple weak or moderate models together can produce stronger results than a single model.

Main types:

  1. Bagging (Bootstrap Aggregating)

    • Models are trained independently on different random subsets of data.

    • Final output is averaged (regression) or majority vote (classification).

    • Example: Random Forest

  2. Boosting

    • Models are trained sequentially.

    • Each new model focuses on correcting errors of previous models.

    • Examples:

      • AdaBoost

      • Gradient Boosting

      • XGBoost

      • LightGBM

  3. Stacking

    • Multiple models are trained.

    • Their outputs are used as inputs to a final meta-model.

Benefits:

  • Reduced variance

  • Reduced bias

  • Better generalization

  • Improved robustness

In deep learning, ensembles can be created by:

  • Training multiple neural networks with different initializations

  • Using different architectures

  • Averaging predictions

Ensemble methods often achieve higher accuracy in real-world machine learning competitions and production systems.

What happens during the forward pass in a neural network?

During the forward pass:

  • Input flows from left to right
  • Each layer computes outputs using current weights
  • Final output is produced

Example

Input image → convolution layers → dense layers → class probabilities

No learning happens here; it’s just prediction using current parameters.


Why do we need a loss function in deep learning?

The loss function measures how wrong the prediction is.

It provides:

  • A numeric error signal
  • Direction for improvement

Example

Actual price = 100

Predicted price = 130

Loss = difference between 100 and 130

Without loss:

  • Model has no idea what “better” means
  • No way to update weights meaningfully

How does the choice of loss function depend on the problem type (classification vs regression)?

Loss functions match the nature of the output.

Regression

  • Output is continuous
  • Common loss: Mean Squared Error (MSE)

Example:

  • Predicting house prices

Classification

  • Output is a class or probability
  • Common loss: Cross-Entropy Loss

Example:

  • Email spam vs not spam

Using the wrong loss leads to:

  • Slow learning
  • Poor predictions
  • Incorrect gradients

What is the difference between Single-Layer and Multi-Layer Perceptron?

Single-Layer Perceptron (SLP):

  • Has only one layer of trainable weights.

  • No hidden layers.

  • Input is directly connected to output.

  • Can solve only linearly separable problems.

Structure:
Input → Output

Limitation:
It cannot solve non-linear problems such as the XOR problem.

Multi-Layer Perceptron (MLP):

  • Has one or more hidden layers.

  • Uses activation functions between layers.

  • Can solve non-linear problems.

  • More powerful and flexible than SLP.

Structure:
Input → Hidden Layer(s) → Output

Key difference:

SLP can model only linear decision boundaries, while MLP can model complex non-linear decision boundaries due to hidden layers and activation functions.

What are the components of a neural network?

The main components of a neural network are:

  1. Input Layer
    Receives input features.
    Example: pixels of an image, numerical features, text embeddings.

  2. Hidden Layers
    Perform intermediate computations and feature extraction.
    Can be multiple layers.

  3. Output Layer
    Produces final prediction.
    Example: class probabilities, regression value.

  4. Weights
    Parameters that determine the strength of connection between neurons.

  5. Bias
    Additional parameter added to shift the activation function.

  6. Activation Function
    Introduces non-linearity.
    Examples: ReLU, sigmoid, tanh, softmax.

  7. Loss Function
    Measures difference between predicted output and true output.
    Examples: MSE, Cross-Entropy.

  8. Optimizer
    Updates weights to minimize loss.
    Examples: Gradient Descent, Adam.

  9. Backpropagation Mechanism
    Computes gradients and adjusts weights using the chain rule.

Explain how a neural network works.

A neural network works in three main stages:

  1. Forward Propagation

Each neuron performs:

Output = Activation(Weighted Sum of Inputs + Bias)

Mathematically:
z = W·x + b
a = activation(z)

Data flows from input layer to output layer.

  1. Loss Computation

The predicted output is compared with the actual target using a loss function.

Example:
Loss = (Predicted − Actual)²

  1. Backpropagation

  • The loss is propagated backward.

  • Gradients of the loss with respect to each weight are computed using the chain rule.

  • Weights are updated using an optimizer.

Update rule (simplified):
New Weight = Old Weight − Learning Rate × Gradient

This process repeats over many iterations (epochs) until the model minimizes loss.

In summary:
Input → Linear Combination → Activation → Loss → Backpropagation → Weight Update

What role do weights and bias play in a neural network? How are the weights initialized?

Role of Weights:

  • Weights determine how strongly one neuron influences another.

  • They control how input features are combined.

  • During training, weights are adjusted to minimize loss.

Role of Bias:

  • Bias shifts the activation function.

  • Allows the model to fit data that does not pass through the origin.

  • Increases flexibility of the model.

Example:

Without bias:
y = wx

With bias:
y = wx + b

Bias helps the model move the decision boundary.

How are weights initialized?

Weights should not be initialized to zero. Instead, they are initialized with small random values.

Common initialization methods:

  1. Random Initialization
    Small random numbers from normal or uniform distribution.

  2. Xavier (Glorot) Initialization
    Used for sigmoid and tanh activations.
    Keeps variance stable across layers.

  3. He Initialization
    Used for ReLU activation.
    Designed to prevent vanishing/exploding gradients.

Proper initialization helps:

  • Faster convergence

  • Stable gradient flow

  • Avoiding vanishing or exploding gradients

Can a neural network be trained by initializing all the weights to 0 or a constant?

No, it should not be trained by initializing all weights to 0 or the same constant.

Reason: Symmetry Problem

If all weights are initialized to the same value:

  • All neurons in a layer will produce the same output.

  • They will receive the same gradient during backpropagation.

  • They will update in the same way.

  • All neurons will remain identical.

This means the network behaves as if each layer has only one neuron, reducing its learning capacity.

Why random initialization is necessary:

Random initialization breaks symmetry between neurons.
Different neurons learn different features.

Important distinction:

Bias can sometimes be initialized to zero.
Weights must be initialized randomly.

Initializing all weights to zero prevents effective learning in multi-layer neural networks.

What is forward and backward propagation?

Forward propagation is the process where input data passes through the neural network to produce an output.

Steps:

  1. Input is fed into the network.

  2. Each layer computes a weighted sum:
    z = W·x + b

  3. Activation function is applied:
    a = activation(z)

  4. This continues layer by layer until final output is produced.

  5. Loss is computed using a loss function.

Backward propagation is the process of computing gradients of the loss with respect to weights and biases and updating them.

Steps:

  1. Compute derivative of loss with respect to output.

  2. Use chain rule to propagate gradients backward through each layer.

  3. Compute gradients for weights and biases.

  4. Update parameters using an optimizer:
    Weight = Weight − Learning Rate × Gradient

Forward pass computes predictions.
Backward pass updates parameters to reduce error.

What are the challenges with gradient descent, and what are the treatments?

Challenges:

  1. Slow convergence
    Especially in large datasets or poorly conditioned surfaces.

  2. Getting stuck in local minima or saddle points
    Loss surface may not be smooth.

  3. Oscillation
    Occurs when gradients are steep in one direction.

  4. Vanishing gradients
    Gradients become too small in deep networks.

  5. Exploding gradients
    Gradients become too large and destabilize training.

Treatments:

  • Use better optimizers like Adam or RMSProp.

  • Use momentum to smooth oscillations.

  • Proper weight initialization (He or Xavier).

  • Normalize inputs and use Batch Normalization.

  • Gradient clipping to handle exploding gradients.

  • Learning rate scheduling.

What is Adaptive Moment Estimation (Adam)?

Adam is an optimization algorithm that combines:

  • Momentum

  • RMSProp

It maintains two moving averages:

  1. First moment (mean of gradients)

  2. Second moment (variance of gradients)

Update steps:

  • Compute gradient.

  • Update moving averages.

  • Correct bias in early steps.

  • Update weights using adaptive learning rate.

Advantages:

  • Fast convergence

  • Handles sparse gradients well

  • Less sensitive to learning rate tuning

  • Works well in large models

Adam adapts the learning rate individually for each parameter

What is the vanishing gradient problem? How is it different from the exploding gradient problem?

Vanishing gradient problem:

During backpropagation in deep networks, gradients become extremely small.

As gradients move backward through many layers, they get multiplied repeatedly. If values are less than 1, gradients shrink exponentially.

Consequences:

  • Early layers learn very slowly.

  • Training becomes ineffective.

  • Common with sigmoid and tanh activations.

Exploding gradient problem:

Gradients grow exponentially during backpropagation.

Consequences:

  • Extremely large weight updates.

  • Unstable training.

  • Loss becomes NaN.

  • Model diverges.

Difference:

Vanishing gradients → gradients approach zero → slow learning
Exploding gradients → gradients become very large → unstable learning

Both problems are common in deep networks and RNNs.


[Image/Media]

What are the ways to treat the vanishing and exploding gradient problem?

For vanishing gradients:

  • Use ReLU activation instead of sigmoid/tanh.

  • Use proper initialization (He or Xavier).

  • Use Batch Normalization.

  • Use residual connections (ResNet).

  • Use LSTM or GRU for sequence models.

For exploding gradients:

  • Gradient clipping.

  • Proper weight initialization.

  • Lower learning rate.

  • Batch normalization.

  • Use regularization.

These techniques stabilize gradient flow and improve deep network training.

What are batch, mini-batch, and stochastic gradient descent? Which one would you use and why?

Batch Gradient Descent:

  • Uses entire dataset to compute gradient.

  • Stable but slow.

  • Not suitable for large datasets.

Stochastic Gradient Descent (SGD):

  • Uses one data point at a time.

  • Fast updates.

  • Noisy and unstable.

Mini-Batch Gradient Descent:

  • Uses small batch of data (e.g., 32, 64, 128).

  • Balanced approach.

  • Faster than batch.

  • More stable than pure SGD.

  • Efficient on GPUs.

In practice, mini-batch gradient descent is preferred because:

  • It balances stability and speed.

  • Efficient for parallel computation.

  • Works well in large datasets.

Epoch:

One complete pass through the entire dataset.

Batch:

A subset of the dataset used in one forward and backward pass.

Iteration:

One update step of the model parameters.

Relationship:

Explain the Learning Rate in a neural network model.

Learning rate controls how much weights are updated during training.

Update rule:

Weight = Weight − Learning Rate × Gradient

If learning rate is too low:

  • Training is very slow.

  • May get stuck in local minima.

  • Requires many epochs.

If learning rate is too high:

  • Training becomes unstable.

  • Loss may oscillate.

  • May diverge.

  • Model may never converge.

Ideal learning rate:

  • Large enough for fast convergence.

  • Small enough to avoid overshooting.

Techniques:

  • Learning rate scheduling

  • Reduce on plateau

  • Cyclical learning rates

  • Warmup strategies

How would you use backpropagation?

Backpropagation is used to train neural networks.

Steps to use backpropagation:

  1. Initialize weights randomly.

  2. Perform forward propagation to compute output.

  3. Compute loss between predicted and actual output.

  4. Compute gradients of loss with respect to each parameter using chain rule.

  5. Update weights using gradient descent or an optimizer.

  6. Repeat for multiple epochs until loss is minimized.

Backpropagation enables the network to learn by systematically reducing prediction error through gradient-based optimization.

What is an activation function, and why is it needed?

An activation function is a mathematical function applied to the output of a neuron after the weighted sum of inputs is computed.

For a neuron:

z = W·x + b
a = activation(z)

The activation function introduces non-linearity into the model.

Why it is needed:

  1. Without activation functions, a neural network becomes a linear model.
    Multiple linear layers collapse into a single linear transformation.

  2. Real-world problems are non-linear.
    Tasks like image recognition, speech recognition, and language modeling require modeling complex relationships.

  3. Activation functions allow the network to learn complex patterns by transforming linear combinations into non-linear mappings.

Common activation functions:

  • Sigmoid

  • Tanh

  • ReLU

  • LeakyReLU

  • Softmax

Which activation function do we use for each layer?

Choice of activation depends on the type of problem and layer.

Hidden layers:

  • ReLU is most commonly used.

  • LeakyReLU is used when avoiding dying ReLU problem.

  • Tanh is sometimes used but less common in deep networks.

Output layer:

For binary classification:

  • Sigmoid

For multi-class classification:

  • Softmax

For regression:

  • No activation (linear output)

For multi-label classification:

  • Sigmoid (one per output unit)

General rule:

  • Hidden layers → ReLU (or variants)

  • Output layer → Depends on task type


[Image/Media]

Why can we not use sigmoid or tanh to activate the hidden layer of the neural network?

Sigmoid and tanh can technically be used in hidden layers, but they create practical problems in deep networks.

Main issues:

  1. Vanishing gradient problem

Sigmoid and tanh saturate for large positive or negative values.
Their derivatives become very small near extremes.

During backpropagation, small gradients get multiplied across many layers, causing gradients to shrink toward zero.

This slows learning in early layers.

  1. Saturation problem

When inputs fall in saturated regions:

  • Output becomes almost constant.

  • Neurons stop learning effectively.

  1. Sigmoid is not zero-centered

Sigmoid outputs values between 0 and 1.

This causes gradients to have consistent signs, leading to inefficient weight updates.

Tanh partially solves this because it is zero-centered, but it still suffers from vanishing gradients.

For deep networks, ReLU works better because:

  • It does not saturate for positive inputs.

  • It has constant gradient (1) for positive region.

  • It trains faster.


[Image/Media]

What is the difference between ReLU and LeakyReLU functions?

ReLU (Rectified Linear Unit):

f(x) = max(0, x)

If x > 0 → output = x
If x ≤ 0 → output = 0

Advantages:

  • Simple and computationally efficient.

  • Reduces vanishing gradient problem.

  • Fast convergence.

Problem:
Dying ReLU problem
If inputs are always negative, neuron outputs 0 and gradient becomes 0.
Neuron stops updating permanently.

LeakyReLU:

f(x) = x if x > 0
f(x) = αx if x ≤ 0

Where α is a small constant (e.g., 0.01).

Difference:

ReLU:

  • Zero gradient for negative inputs.

LeakyReLU:

  • Small non-zero gradient for negative inputs.

  • Prevents neurons from dying completely.

LeakyReLU improves gradient flow in negative region and reduces the dying ReLU issue.


[Image/Media]

How can a Tanh activation function help the neural network converge faster than a logistic sigmoid?

Tanh outputs values between -1 and 1.

Sigmoid outputs values between 0 and 1.

Key advantage of tanh:

Zero-centered output

Since tanh outputs are centered around zero, the gradients during backpropagation are more balanced.

With sigmoid:

  • Outputs are always positive.

  • Gradients tend to have the same sign.

  • This can cause inefficient zig-zag updates during optimization.

With tanh:

  • Outputs can be positive or negative.

  • Gradients are more symmetric.

  • Optimization becomes more stable.

As a result, networks using tanh often converge faster than those using sigmoid.

However, tanh still suffers from vanishing gradients in deep networks, which is why ReLU is generally preferred in modern deep learning.

How do you differentiate between CNN and RNN? Which algorithm should be used when?

CNN (Convolutional Neural Network):

  • Designed for spatial data.

  • Uses convolution operations.

  • Captures local patterns using filters.

  • Parallel computation possible.

  • Best suited for image and grid-like data.

Applications:

  • Image classification

  • Object detection

  • Medical image analysis

  • Video frame analysis

RNN (Recurrent Neural Network):

  • Designed for sequential data.

  • Has recurrent connections (remembers previous inputs).

  • Maintains hidden state.

  • Processes data step by step in sequence.

Applications:

  • Language modeling

  • Machine translation

  • Speech recognition

  • Time-series forecasting

Key differences:

CNN:

  • Captures spatial dependencies.

  • No memory of previous inputs.

  • Efficient parallel training.

RNN:

  • Captures temporal dependencies.

  • Maintains memory via hidden state.

  • Sequential processing (slower).

When to use:

  • Use CNN for images, spatial data, or grid-like structures.

  • Use RNN (or LSTM/GRU/Transformers) for sequences and time-dependent data.

Why do we use CNN for image data and not Feedforward Neural Network (FNN)?

Feedforward Neural Network:

  • Fully connected layers.

  • Every input neuron connects to every neuron in next layer.

  • Large number of parameters.

  • Does not preserve spatial structure.

For example:

A 256×256 RGB image has:
256 × 256 × 3 = 196,608 input values.

If connected to 1,000 neurons in the next layer:
196,608 × 1,000 = nearly 197 million parameters.

Problems with FNN for images:

  • Too many parameters.

  • High memory usage.

  • Overfitting risk.

  • No spatial feature extraction.

  • Ignores local patterns.

CNN advantages:

  1. Local connectivity
    Filters focus on small regions (e.g., 3×3).

  2. Parameter sharing
    Same filter slides across the image.

  3. Preserves spatial structure
    Detects edges, textures, shapes.

  4. Efficient training
    Far fewer parameters than FNN.

CNN is specifically designed for image data, while FNN is not efficient for high-dimensional spatial inputs.

Conv1D:

  • Used for 1-dimensional data.

  • Filter moves in one direction.

  • Example: time series, audio signals, text sequences.

  • Used for 2-dimensional data.

  • Filter moves across height and width.

  • Example: images.

Input shape example:

  • Filter moves across height, width, and depth.

Input shape example:
(depth, height, width, channels)

Difference summary:

Conv1D → sequential data
Conv2D → images

Explain the different layers of CNN. When using CNN, which layers are the parameters estimated for?

Main layers of CNN:

  1. Convolutional Layer
    Applies filters to extract features.
    Learns weights of filters.

  2. Activation Layer
    Applies non-linearity (usually ReLU).

  3. Pooling Layer
    Reduces spatial dimensions.
    No learnable parameters.

  4. Batch Normalization (optional)
    Normalizes activations.
    Has learnable scale and shift parameters.

  5. Fully Connected (Dense) Layer
    Final classification or regression.
    Has learnable weights and biases.

  6. Dropout
    Regularization technique.
    No learnable parameters.

Parameters are estimated for:

  • Convolutional layers (filter weights and biases).

  • Dense (fully connected) layers.

  • Batch normalization (scale and shift).

Pooling and activation layers do not have trainable parameters.

What is the difference between valid and same padding in CNN?

Padding refers to adding extra pixels (usually zeros) around input before convolution.

Valid padding:

  • No padding is added.

  • Output size is smaller than input.

  • Information near edges may be reduced.

Output size formula:
Output = (Input − Kernel + 1)

Same padding:

  • Padding added so output size equals input size (for stride = 1).

  • Preserves spatial dimensions.

  • Useful when stacking many convolution layers.

Difference:

Valid:

  • Output shrinks.

  • No artificial pixels added.

Same:

  • Output size maintained.

  • Zeros added around borders.

What is pooling? How does it help in filtering features?

Pooling is a downsampling operation that reduces spatial dimensions of feature maps.

Common types:

  1. Max Pooling
    Takes maximum value in a region.

  2. Average Pooling
    Takes average value in a region.

Example:

For a 2×2 region:
[1, 5
2, 3]

Max pooling → 5
Average pooling → 2.75

Benefits:

  1. Reduces dimensionality
    Fewer parameters in next layers.

  2. Reduces computation
    Faster training.

  3. Controls overfitting
    Simplifies representation.

  4. Translation invariance
    Small shifts in image do not drastically change output.

Pooling helps retain important features while discarding less important details.

How can a Dense Layer of a CNN convert into a Fully Convolutional Layer?

A Dense layer performs:

Output = W·x + b

A Fully Convolutional Layer can replicate this using a convolution operation.

Conversion idea:

A Dense layer can be replaced by a convolution layer with:

  • Kernel size equal to input feature map size.

  • Same number of filters as number of output neurons.

Example:

If feature map size is 7×7×512 and Dense layer outputs 1,000 neurons:

Replace Dense with:
Conv2D with:

  • Kernel size = 7×7

  • Filters = 1,000

This produces output of size 1×1×1000, equivalent to Dense output.

Why convert to Fully Convolutional Network (FCN)?

  • Allows variable input image sizes.

  • Useful in segmentation tasks.

  • Preserves spatial information.

Fully Convolutional Networks are widely used in semantic segmentation and object detection.

What is data or image augmentation?

Data augmentation is a technique used to artificially increase the size and diversity of a training dataset by applying transformations to existing data.

In image augmentation, common transformations include:

  • Rotation

  • Flipping (horizontal/vertical)

  • Cropping

  • Scaling

  • Translation (shifting)

  • Adding noise

  • Brightness or contrast adjustment

Example:

What is the difference between dropout and batch normalization?

Dropout:

  • A regularization technique.

  • Randomly sets a fraction of neurons to zero during training.

  • Prevents neurons from co-adapting.

  • Reduces overfitting.

  • Active only during training, not during inference.

Batch Normalization:

  • Normalizes activations within a mini-batch.

  • Reduces internal covariate shift.

  • Stabilizes and accelerates training.

  • Has learnable scale and shift parameters.

  • Used during both training and inference (with running statistics).

Key differences:

Dropout:

  • Reduces overfitting by randomly removing neurons.

  • Introduces randomness.

  • No learnable parameters.

Batch Normalization:

  • Improves training stability.

  • Normalizes intermediate activations.

  • Has learnable parameters.

  • Can also provide slight regularization effect.

Dropout mainly addresses overfitting.
Batch normalization mainly stabilizes and speeds up training.

Does the problem of overfitting happen in neural networks? If yes, then how would you treat it?

Yes, overfitting commonly occurs in neural networks, especially deep networks with many parameters.

Overfitting happens when:

  • Model performs very well on training data.

  • Model performs poorly on unseen test data.

  • Model memorizes noise instead of learning general patterns.

Reasons:

  • High model capacity.

  • Small dataset.

  • Too many training epochs.

  • Complex architecture.

Ways to treat overfitting:

  1. Data augmentation

  2. Dropout

  3. L1 or L2 regularization

  4. Early stopping

  5. Reduce model complexity

  6. Collect more data

  7. Batch normalization

  8. Cross-validation

The goal is to improve generalization performance on unseen data.

Do we have regularizers for neural networks?

Yes, neural networks use several regularization techniques to prevent overfitting.

Common regularizers:

  1. L1 Regularization
    Adds absolute value of weights to loss function.

    Penalizes large weights.

  2. Early Stopping
    Stop training when validation loss increases.

  3. Data Augmentation

  4. Batch Normalization
    Provides some regularization effect.

What is Bottleneck in Autoencoders? Why is it used?

In an autoencoder, the bottleneck is the middle hidden layer with the smallest number of neurons.

Structure:

Input → Encoder → Bottleneck → Decoder → Output

The bottleneck forces the network to compress input data into a lower-dimensional representation.

Why it is used:

  1. Feature learning
    Forces the model to learn the most important features.

  2. Dimensionality reduction
    Acts similar to PCA but can capture non-linear relationships.

  3. Noise removal
    In denoising autoencoders, bottleneck helps ignore noise.

  4. Prevent trivial copying
    Without a bottleneck, the network could simply memorize input and copy it to output.

The bottleneck ensures that the model learns meaningful and compact representations rather than just copying the input directly.

What is transfer learning? Why is it needed? Name commonly used transfer learning models.

Transfer learning is a technique where a model trained on one task (usually large-scale data) is reused for a different but related task.

Instead of training a deep neural network from scratch, we take a pre-trained model and adapt it to our specific problem.

Why it is needed:

  1. Limited data
    Training deep networks from scratch requires very large datasets. Many real-world problems do not have enough labeled data.

  2. Reduced training time
    Pre-trained models have already learned low-level and high-level features.

  3. Better performance
    Models trained on massive datasets (like ImageNet) learn strong general features that transfer well.

  4. Reduced computational cost
    Saves time and GPU resources.

How it works:

  • Load a pre-trained model.

  • Freeze some early layers.

  • Replace final classification layer.

  • Train on new dataset.

Commonly used transfer learning models:

For image tasks:

  • VGG16 / VGG19

  • ResNet (ResNet50, ResNet101)

  • Inception

  • MobileNet

  • EfficientNet

  • DenseNet

For NLP:

  • BERT

  • GPT

  • RoBERTa

  • T5

For vision transformers:

  • ViT

  • Swin Transformer

Transfer learning is especially powerful in computer vision and NLP.

Define Fine-tuning. Why is it used in Deep Learning?

Fine-tuning is the process of taking a pre-trained model and training some or all of its layers on a new dataset.

Difference between transfer learning and fine-tuning:

Transfer learning:

  • Usually freeze most layers.

  • Train only final layers.

Fine-tuning:

  • Unfreeze some deeper layers.

  • Continue training entire or partial network with a small learning rate.

Why fine-tuning is used:

  1. Adaptation to new domain
    Pre-trained model may not perfectly match target dataset.

  2. Improve accuracy
    Allows model to adjust learned features to new task.

  3. Domain-specific feature learning
    For example:

    • Pretrained on general objects.

    • Fine-tuned for medical imaging.

Important practice:

  • Use smaller learning rate during fine-tuning.

  • Unfreeze layers gradually.

Fine-tuning allows the model to retain general knowledge while adapting to new data.
