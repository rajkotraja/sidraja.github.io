Foundations and Attention Basics

Describe the Transformer architecture and how a forward pass produces an output?

Definition

A Transformer is a neural network architecture used mainly for sequence tasks (like text) that relies entirely on attention mechanisms instead of recurrence (RNNs) or convolution (CNNs). It processes all tokens in parallel and learns relationships between them using self-attention.

Why It Is Used

  • Handles long-range dependencies better than RNNs

  • Enables parallel computation → faster training

  • Forms the backbone of modern models like GPT and BERT

  • Works well for tasks like translation, summarization, and question answering

Key Components

  • Embedding Layer: Converts tokens into vectors

  • Positional Encoding: Adds order information to tokens

  • Self-Attention (Multi-Head Attention): Captures relationships between tokens

  • Feed-Forward Network (FFN): Applies non-linear transformation to each token

  • Residual Connections + Layer Normalization: Stabilize training

  • Encoder / Decoder Stacks (depending on model type)

How It Works (Forward Pass Step-by-Step)

1. Input Processing

  • Text is tokenized (e.g., “I love AI” → tokens)

  • Each token is converted into an embedding vector

  • Positional encoding is added to preserve order

2. Self-Attention Mechanism

For each token:

  • Create Query (Q), Key (K), Value (V) vectors

  • Compute attention scores: how much each token attends to others

  • Apply softmax to normalize scores

  • Multiply scores with Value vectors → weighted sum

3. Multi-Head Attention

  • Run multiple attention operations in parallel

  • Each head learns different relationships

  • Concatenate outputs and project back

4. Add & Normalize

  • Add residual connection (input + attention output)

  • Apply layer normalization

5. Feed-Forward Network

  • Apply a small neural network to each token independently

  • Adds non-linearity and richer representations

6. Repeat Layers

  • Steps 2–5 are stacked multiple times (e.g., 12 layers)

7. Output Layer

  • Final representations are passed to a task-specific layer

    • Classification → softmax

    • Generation → probability distribution over next token

Advantages

  • Parallel processing → faster than RNNs

  • Captures global context effectively

  • Scales well with large datasets

  • Flexible for many tasks

Disadvantages

  • High computational cost (especially for long sequences)

  • Requires large amounts of data

  • Attention complexity is quadratic with sequence length

Simple Real-World Example

Sentence: “The cat sat on the mat”

When processing the word “sat,” the Transformer can attend to “cat” to understand who is performing the action, even though they are separated by other words. This helps it build better contextual meaning.

Interview-Ready Answer

A Transformer is a neural network architecture that uses self-attention to process sequences in parallel, replacing recurrence and convolution. During a forward pass, input tokens are embedded and combined with positional encodings, then passed through stacked layers of multi-head self-attention and feed-forward networks with residual connections and normalization. The attention mechanism computes relationships between all tokens using query, key, and value vectors, producing contextualized representations that are finally used for prediction tasks like classification or text generation.

Define a Large Language Model (LLM) in your words and outline how it generates text?

Definition

A Large Language Model (LLM) is a type of neural network trained on massive amounts of text data to understand and generate human-like language. In simple terms, it learns patterns in language so it can predict what word (or token) should come next in a sequence.

Why It Is Used

  • Generates natural, coherent text

  • Understands context and meaning in language

  • Powers applications like chatbots, translation, summarization, and coding assistants

  • Reduces the need for task-specific models

Core Mechanism (How It Generates Text)

1. Tokenization

  • Input text is broken into smaller units called tokens (words/subwords)

  • Example: “Hello world” → ["Hello", "world"]

2. Embedding

  • Each token is converted into a numerical vector representing meaning

3. Context Understanding (Transformer Layers)

  • Tokens pass through multiple layers of self-attention

  • The model learns relationships between words (context)

  • Example: In “bank of river” vs “bank loan,” it understands different meanings

4. Next-Token Prediction

  • The model outputs probabilities for the next possible token

  • It selects one (highest probability or sampled choice)

5. Iterative Generation

  • The predicted token is added to the input

  • Steps repeat to generate full sentences

Key Components

  • Transformer Architecture (attention-based model)

  • Self-Attention Mechanism (captures relationships between tokens)

  • Training Objective: Predict the next token

  • Large Dataset + Parameters (millions/billions of weights)

Advantages

  • Produces fluent, human-like text

  • Handles a wide variety of language tasks

  • Learns general language patterns (not task-specific)

Disadvantages

  • Requires huge computational resources

  • Can generate incorrect or misleading information

  • May reflect biases from training data

Simple Real-World Example

Input: “The sky is”

  • The LLM predicts likely next words like “blue,” “clear,” or “falling”

  • If “blue” is chosen, it continues:
    “The sky is blue and…”

Interview-Ready Answer

A Large Language Model is a neural network trained on large text datasets to understand and generate human language by predicting the next token in a sequence. It works by converting input text into embeddings, processing them through Transformer layers with self-attention to capture context, and then outputting probability distributions over possible next tokens. By repeatedly predicting and appending tokens, it generates coherent text.

List the major building blocks of a Transformer and briefly state each role?

Major Building Blocks of a Transformer (and their roles)

1. Tokenization

Breaks input text into smaller units called tokens (words or subwords).
Role: Converts raw text into manageable pieces for the model to process.

2. Embedding Layer

Maps each token into a dense numerical vector.
Role: Represents words in a way that captures semantic meaning.

3. Positional Encoding

Adds position information to token embeddings.
Role: Since Transformers don’t process sequentially, this tells the model the order of words.

4. Self-Attention Mechanism

Computes relationships between all tokens in a sequence using Query, Key, and Value vectors.
Role: Helps the model understand context by deciding which words are important to each other.

5. Multi-Head Attention

Runs multiple self-attention operations in parallel.
Role: Allows the model to capture different types of relationships (syntax, meaning, long-range dependencies).

6. Feed-Forward Neural Network (FFN)

A small fully connected neural network applied to each token independently.
Role: Adds non-linearity and transforms learned representations.

7. Residual Connections

Skip connections that add input back to output of sublayers.
Role: Helps gradient flow and stabilizes deep networks.

8. Layer Normalization

Normalizes outputs within each layer.
Role: Improves training stability and speeds up convergence.

9. Encoder Stack (optional in some models)

A stack of repeated layers containing attention + FFN.
Role: Builds rich contextual representations of the input.

10. Decoder Stack (in generative models like GPT)

Similar to encoder but includes masked attention.
Role: Generates output step-by-step while preventing looking ahead.

11. Output Layer (Linear + Softmax)

Converts final hidden states into probability distribution over vocabulary.
Role: Predicts the next token.

Interview-Ready Answer

A Transformer is made up of tokenization, embeddings, positional encoding, self-attention, multi-head attention, feed-forward networks, residual connections, layer normalization, and stacked encoder/decoder blocks, followed by an output layer. Together, these components allow the model to convert text into vectors, capture contextual relationships using attention, transform representations through deep layers, and finally predict the next token in a sequence.

Explain self-attention and how a token attends to others within a Transformer layer?

Self-Attention Mechanism (Definition)

Self-attention is a mechanism in Transformers that allows each token in a sequence to look at all other tokens and decide which ones are most relevant to it. It helps the model build context-aware representations of words.

In simple terms:
Each word asks, “Which other words in this sentence matter to me, and how much?”

Why It Is Used

  • Captures relationships between words regardless of distance

  • Understands context dynamically (same word can mean different things)

  • Works in parallel (unlike sequential RNNs)

  • Enables long-range dependency learning

Key Idea

For each token, the model creates three vectors:

  • Query (Q) → what this token is looking for

  • Key (K) → what each token offers

  • Value (V) → actual information carried by each token

Step-by-Step: How a Single Token Attends to Others

Assume sentence:
“The cat sat on the mat”

We focus on the token: “sat”

1. Convert Tokens into Vectors

Each word is converted into embeddings:

  • “The” → vector

  • “cat” → vector

  • “sat” → vector (our focus token)

  • “on” → vector

  • “the” → vector

  • “mat” → vector

2. Create Q, K, V for Each Token

For every token:

  • Q = Query vector

  • K = Key vector

  • V = Value vector

For “sat”:

  • Q_sat is generated

  • It will compare against all K vectors

3. Compute Attention Scores

We compare Q_sat with every K:

  • score(sat, The) = Q_sat · K_The

  • score(sat, cat) = Q_sat · K_cat

  • score(sat, sat) = Q_sat · K_sat

  • score(sat, on) = Q_sat · K_on

  • score(sat, mat) = Q_sat · K_mat

This dot product measures relevance.

4. Scale and Normalize

  • Scores are scaled (to stabilize values)

  • Softmax is applied

This produces probabilities:

  • “cat” → high weight

  • “mat” → medium weight

  • “the” → low weight

So the model learns what matters most for “sat”.

5. Weighted Sum of Values

Now we combine information:

[
\text{Output}{sat} = \sum (\text{attention weight} \times V{token})
]

So:

  • Strongly weighted tokens contribute more

  • Weakly related tokens contribute less

Result:
“sat” becomes a context-rich representation influenced by “cat” (subject) and “mat” (object context).

6. Multi-Head Extension (Important Add-on)

Instead of one attention calculation:

  • Multiple heads run in parallel

  • Each head learns different relationships:

    • One focuses on grammar

    • One focuses on subject-object relations

    • One focuses on long-range dependencies

Outputs are combined for a richer understanding.

Key Insight

Self-attention transforms each token from a fixed meaning into a context-aware meaning by dynamically mixing information from all other tokens.

Advantages

  • Understands context deeply

  • Handles long sentences well

  • Captures relationships regardless of distance

Disadvantages

  • Computationally expensive (O(n²) complexity)

  • Memory intensive for long sequences

Interview-Ready Answer

Self-attention is a mechanism where each token in a Transformer computes its relevance to every other token in the sequence using Query, Key, and Value vectors. For a given token, its query is compared with all keys to compute attention scores, which are normalized using softmax to form weights. These weights are used to take a weighted sum of all value vectors, producing a context-aware representation of the token. This allows each word to dynamically focus on the most relevant parts of the sentence, capturing relationships regardless of distance.

What roles do Query, Key, Value play in attention, and how do they interact mathematically?

Role of Query, Key, and Value in Attention (Intuition)

In self-attention, every token is projected into three different vectors:

  • Query (Q) → what this token is looking for

  • Key (K) → what each token offers (its “label” or identity for matching)

  • Value (V) → the actual information content to be passed along

A simple analogy:

  • Query = a search request

  • Key = index labels in a database

  • Value = the actual data stored

So attention is essentially:
“Match Query with Keys to decide how much Value to take from each token.”

How They Interact Mathematically

Assume we have a sequence of token embeddings stacked into matrices:

  • ( X \in \mathbb{R}^{n \times d} )

We compute:

1. Linear projections

Each token embedding is transformed:

  • ( Q = XW_Q )

  • ( K = XW_K )

  • ( V = XW_V )

Where:

  • ( W_Q, W_K, W_V ) are learned weight matrices

2. Compute similarity scores

We measure how much each token should attend to others:

[
\text{Score} = QK^T
]

  • Each row of Q is compared with all rows of K

  • Produces an ( n \times n ) matrix of attention scores

3. Scale scores (stability step)

[
\text{Scaled Score} = \frac{QK^T}{\sqrt{d_k}}
]

  • Prevents values from becoming too large

  • Improves gradient stability

4. Convert to probabilities (attention weights)

[
A = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)
]

  • Each row sums to 1

  • Represents how much each token attends to others

5. Weighted sum of values

[
\text{Output} = AV
]

  • Each token’s output is a weighted combination of all Value vectors

  • Weights come from attention matrix A

Step-by-Step Flow for One Token

For a single token (say “sat”):

  1. Take its Query vector ( q_{sat} )

  2. Compare with all Keys:
    [
    q_{sat} \cdot k_i
    ]

  3. Normalize scores using softmax

  4. Use weights to combine all Value vectors:
    [
    \sum_i \alpha_i v_i
    ]

Result: a new representation of “sat” enriched with context.

Key Insight

  • Query decides what to look for

  • Key decides how matchable a token is

  • Value provides the actual information passed forward

Attention is essentially:

similarity(Q, K) → weights → weighted sum of V

Advantages of This Design

  • Flexible context modeling

  • Parallel computation

  • Learns relationships dynamically instead of fixed rules

Interview-Ready Answer

In attention mechanisms, each token is projected into Query, Key, and Value vectors. The Query represents what a token is searching for, the Key represents the identity used for matching, and the Value contains the actual information. Mathematically, attention is computed by taking the dot product of Query and Key ((QK^T)), scaling it, applying softmax to get attention weights, and then multiplying these weights with Value vectors to produce the final output. This allows each token to selectively aggregate relevant information from all other tokens in the sequence.

Why does multi-head attention exist? What advantages do multiple heads provide?

Why Multi-Head Attention Exists (Core Idea)

Multi-head attention exists because a single attention operation is too limited to capture all types of relationships in a sequence at once.

Instead of relying on one “view” of relationships, the model uses multiple parallel attention heads, each learning a different representation of the same input.

What Problem It Solves

If you use only one attention head:

  • It tends to focus on a single dominant type of relationship

  • It may miss other important patterns (syntax, semantics, long-range links)

  • It compresses all relational learning into one space → bottleneck

Multi-head attention removes this bottleneck.

How It Works (Mechanically)

Given input embeddings (X):

Each head independently computes:

  • ( Q_i = XW_Q^{(i)} )

  • ( K_i = XW_K^{(i)} )

  • ( V_i = XW_V^{(i)} )

Then each head performs attention:

[
\text{head}_i = \text{Attention}(Q_i, K_i, V_i)
]

Finally:

  • All heads are concatenated:
    [
    \text{Concat}(\text{head}_1, \text{head}_2, ..., \text{head}_h)
    ]

  • Then projected:
    [
    \text{Output} = \text{Concat}(\cdot) W_O
    ]

Why Multiple Heads Help (Advantages)

1. Different Heads Learn Different Relationships

Each head specializes in a different pattern:

  • One head may focus on syntax (subject–verb agreement)

  • Another may focus on semantic similarity

  • Another may track long-distance dependencies

2. Multiple Representation Subspaces

Each head operates in a different learned subspace of the same input.

This means:

  • The model does not rely on a single perspective

  • It builds a richer, more diverse understanding

3. Better Long-Range Dependency Modeling

Some heads learn:

  • Local relationships (nearby words)

  • Others learn global relationships (far-apart tokens)

This combination improves context understanding.

4. Reduces Information Bottleneck

Instead of compressing all reasoning into one attention map:

  • Each head processes a portion of the “attention capacity”

This improves expressiveness.

5. Improves Robustness

If one head fails to capture a pattern:

  • Other heads may still capture it correctly

  • Makes the model more stable and reliable

Simple Intuition

Think of a sentence as a movie scene:

Single-head attention = one person watching and trying to notice everything
Multi-head attention = multiple viewers:

  • One watches actions

  • One watches relationships

  • One watches background context

  • One tracks long-term story flow

Then their observations are combined.

Key Trade-off

Advantages:

  • Richer representations

  • Better generalization

  • Stronger context modeling

Cost:

  • More computation and memory

  • More parameters

Interview-Ready Answer

Multi-head attention exists to allow the model to capture different types of relationships in parallel. Instead of computing a single attention map, the input is projected into multiple Query, Key, and Value sets, and each head independently computes attention in its own subspace. The outputs are concatenated and linearly transformed. This enables different heads to specialize in different patterns such as syntax, semantics, and long-range dependencies, resulting in richer and more expressive representations compared to single-head attention.

Compare "attention" vs "self-attention"—when are they the same and when do they differ?

Definition: “Attention” (General)

Attention is a broad mechanism where a model learns to focus on relevant parts of some input while producing an output.

It generally involves:

  • A Query (what we are looking for)

  • Keys (what we compare against)

  • Values (information we retrieve)

But crucially, the Query does not have to come from the same source as Keys and Values.

Definition: “Self-Attention”

Self-attention is a specific type of attention where:

Queries, Keys, and Values all come from the same input sequence.

So the model is essentially:

“Let each token look at all other tokens in the same sentence.”

Key Relationship

  • Attention = umbrella concept

  • Self-attention = one special case of attention

When They Are the Same

They are the same when:

  • Query, Key, and Value all come from the same sequence

Example:

  • Sentence: “The cat sat on the mat”

  • Each word attends to all other words in the same sentence

This is self-attention, which is also a form of attention.

So in Transformer encoders:

  • “attention” often implicitly means self-attention

When They Differ

They differ when Query comes from a different source than Key/Value.

1. Cross-Attention (Encoder-Decoder Models)

Used in translation models.

Example:

  • Query: decoder state (“what am I generating now?”)

  • Key/Value: encoder output (“input sentence representation”)

So:

  • Decoder asks: “Which parts of the input sentence matter for this output token?”

This is attention but NOT self-attention.

2. Retrieval-style Attention

Example:

  • Query: user question embedding

  • Keys/Values: database of documents

Again:

  • Different sources → general attention, not self-attention

Side-by-Side Comparison

Self-Attention

  • Q = K = V come from same sequence

  • Used within encoder or decoder

  • Models internal relationships in a sentence

General Attention

  • Q comes from one source

  • K and V may come from another source

  • Used in encoder-decoder and retrieval systems

Intuition

  • Self-attention: “How do words in this sentence relate to each other?”

  • Attention (general): “Given this query, which external information should I use?”

Why the Distinction Matters

  • Self-attention builds context inside a sequence

  • Cross-attention connects two different sequences or systems

Modern Transformers often combine both:

  • Encoder → self-attention

  • Decoder → self-attention + cross-attention

Interview-Ready Answer

Attention is a general mechanism where a model uses a query to selectively focus on relevant parts of key-value pairs. Self-attention is a special case of attention where the query, key, and value all come from the same input sequence, allowing each token to attend to every other token in that sequence. They differ when the query comes from a different source than the keys and values, such as in encoder-decoder architectures, where cross-attention is used to relate output tokens to an encoded input sequence.

Walk through Bahdanau attention and why it was a turning point in sequence modeling?

Definition: Bahdanau Attention

Bahdanau attention (also called additive attention) is an attention mechanism introduced to improve sequence-to-sequence (Seq2Seq) models by allowing the decoder to dynamically focus on different parts of the input sequence at each output step.

It was introduced in neural machine translation to replace the fixed-length “context vector” bottleneck of RNN-based encoder–decoder models.

Why It Was Needed (Problem It Solved)

Before Bahdanau attention (classic Seq2Seq with RNNs):

  • Encoder reads the entire input sentence

  • Compresses it into a single fixed-size vector

  • Decoder generates output from that one vector

Problem:

  • Long sentences → information loss

  • Hard to retain early-word context

  • Poor translation quality for long sequences

This was the information bottleneck problem.

Core Idea of Bahdanau Attention

Instead of compressing everything into one vector:

The decoder looks at all encoder hidden states and decides which ones matter for each output word.

So:

  • No single fixed context vector

  • Context is computed dynamically at each decoding step

How It Works (Step-by-Step)

Assume:

  • Input sentence is encoded into hidden states:
    [
    h_1, h_2, ..., h_n
    ]

  • Decoder is generating output step-by-step

At decoding step ( t ):

1. Compute Alignment Scores

For each encoder hidden state ( h_i ), compute how relevant it is to decoder state ( s_{t-1} ):

[
e_{t,i} = v^T \tanh(W_1 s_{t-1} + W_2 h_i)
]

This is the additive scoring function.

Key idea:

  • Combines decoder state + encoder state

  • Passes through a small neural network

2. Convert Scores to Probabilities

[
\alpha_{t,i} = \text{softmax}(e_{t,i})
]

  • These are attention weights

  • They sum to 1

  • Indicate importance of each input word

3. Compute Context Vector

[
c_t = \sum_{i=1}^{n} \alpha_{t,i} h_i
]

  • Weighted sum of encoder hidden states

  • Represents “what the decoder should focus on now”

4. Generate Output

Decoder uses:

  • Previous state ( s_{t-1} )

  • Context vector ( c_t )

To produce:

  • Next hidden state ( s_t )

  • Next output word

Intuition (Simple View)

When generating each word:

  • The model “looks back” at the entire input sentence

  • Decides which words matter most right now

  • Builds a focused summary dynamically

Why It Was a Turning Point

1. Removed Fixed-Vector Bottleneck

Before:

  • One vector for entire sentence

After Bahdanau:

  • Different context for every output step

This massively improved long-sequence handling.

2. Enabled Variable Alignment

The model learned:

  • Word-to-word alignment automatically
    Example:

  • Translating “chien” → focuses on “dog”

This made translation interpretable.

3. Improved Long Sentence Performance

  • Better handling of long inputs

  • Reduced information loss

  • Major jump in translation quality

4. Foundation for Modern Attention Models

Bahdanau attention directly influenced:

  • Luong attention (simplified scoring)

  • Transformer self-attention

  • GPT/BERT architectures

It is the conceptual bridge from RNNs to Transformers.

Key Limitation

  • Still relies on RNNs (sequential processing)

  • Slower than Transformer self-attention

  • Less parallelizable

Interview-Ready Answer

Bahdanau attention is an additive attention mechanism introduced in sequence-to-sequence models to overcome the fixed-length context vector limitation of RNN encoder–decoder architectures. Instead of compressing the input into a single vector, it computes alignment scores between the decoder state and each encoder hidden state, normalizes them using softmax to obtain attention weights, and produces a context vector as a weighted sum of encoder states. This allows the decoder to dynamically focus on different parts of the input at each time step. It was a turning point because it eliminated the information bottleneck problem, significantly improved long-sequence modeling, and laid the foundation for modern attention-based architectures like Transformers.

Describe scaled dot-product attention and why divide scores by √d_k?

Definition: Scaled Dot-Product Attention

Scaled dot-product attention is the core attention mechanism used in Transformers. It computes how much each token should attend to others by taking dot products between Query (Q) and Key (K) vectors, scaling the result, and using it to weight Value (V) vectors.

Core Formula

\text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V

Where:

  • (Q) = Queries

  • (K) = Keys

  • (V) = Values

  • (d_k) = dimension of Key vectors

Step-by-Step Working

1. Compute similarity scores

Each Query is compared with all Keys:

[
QK^T
]

  • Gives raw similarity scores

  • Higher value → more relevant token

2. Scale the scores

Divide by ( \sqrt{d_k} ):

[
\frac{QK^T}{\sqrt{d_k}}
]

3. Apply softmax

Converts scores into probabilities:

  • Ensures all weights sum to 1

  • Highlights most relevant tokens

4. Weighted sum of Values

Final output:

  • Each Value vector is weighted by attention scores

  • Produces context-aware representation

Why Do We Divide by ( \sqrt{d_k} )?

Problem without scaling

When (d_k) is large:

  • Dot products (Q \cdot K) become large in magnitude

  • This pushes softmax into extreme regions

Result:

  • Very peaky distributions (almost 0 or 1)

  • Weak gradients during backpropagation

  • Slower or unstable training

Intuition

Dot product magnitude grows with vector dimension:

  • If each component has variance ≈ 1

  • Then dot product variance ≈ (d_k)

So as (d_k) increases:

  • Scores become too large

Solution: normalization by √d_k

Dividing by:

[
\sqrt{d_k}
]

  • Keeps variance of scores roughly stable (~1)

  • Prevents softmax saturation

  • Ensures smoother gradients

Simple Analogy

Without scaling:

  • It’s like shouting comparisons → one option dominates too early

With scaling:

  • It’s like balanced comparisons → softmax can make nuanced decisions

Why This Matters in Transformers

  • Enables stable training of deep attention layers

  • Works well even with large embedding sizes

  • Critical for multi-head attention (each head has smaller (d_k))

Advantages of Scaled Dot-Product Attention

  • Computationally efficient (matrix multiplication friendly)

  • Stable gradients due to scaling

  • Works well in parallel architectures

  • Forms backbone of Transformer models

Interview-Ready Answer

Scaled dot-product attention computes attention scores by taking the dot product between Query and Key vectors, scaling them by dividing by the square root of the Key dimension, applying softmax, and using the result to weight Value vectors. The division by √d_k is important because without scaling, large dimensional dot products produce very large values, causing softmax to saturate and leading to unstable gradients. The scaling keeps variance stable, ensures smoother optimization, and enables effective training of deep Transformer models.

Distinguish attention scores vs weights—when do scores become weights?

Key Idea

In attention, scores and weights are not the same thing.
They represent two different stages of the same process.

1. Attention Scores (Raw Compatibility)

Definition

Attention scores measure how well a Query matches a Key.

Formula

\text{scores} = \frac{QK^T}{\sqrt{d_k}}

What they represent

  • Raw similarity between tokens

  • Unnormalized values

  • Can be positive or negative

  • Not yet interpretable as probabilities

Intuition

“How relevant is token A to token B?”

2. Attention Weights (Normalized Importance)

Definition

Attention weights are the normalized version of attention scores after applying softmax.

Formula

\alpha = \text{softmax}(\text{scores})

What they represent

  • Probability distribution over tokens

  • Non-negative values

  • Sum to 1 across all tokens

  • Used to mix Value vectors

Intuition

“Out of all tokens, how much should I focus on each one?”

3. When Do Scores Become Weights?

Attention scores become attention weights at this step:

Conversion step:

  1. Compute raw scores:
    [
    QK^T / \sqrt{d_k}
    ]

  2. Apply softmax:
    [
    \text{softmax}(\cdot)
    ]

? The moment softmax is applied, scores become weights.

4. Why This Transformation Is Needed

Scores alone are not usable because:

  • They are unbounded

  • Not normalized

  • Cannot be interpreted probabilistically

Softmax fixes this by:

  • Scaling values into (0, 1)

  • Making them sum to 1

  • Emphasizing relative importance

Define causal masking and explain when and why it is applied during attention?

Definition: Causal Masking

Causal masking (also called look-ahead masking) is a technique in Transformer attention that prevents a token from attending to future tokens in a sequence. Each token is only allowed to attend to itself and previous tokens, ensuring a left-to-right flow of information.

Where It Is Applied

Causal masking is applied in:

  • Decoder-only Transformers (e.g., GPT models)

  • Decoder part of encoder–decoder Transformers

It is applied inside the self-attention mechanism, specifically before the softmax step on the attention scores.

How It Works

First, attention scores are computed:

\text{scores} = \frac{QK^T}{\sqrt{d_k}}

Then a causal mask is applied:

  • Upper triangular part of the score matrix (future positions) is set to −∞

  • Lower triangular part (past and current positions) is kept unchanged

After masking, softmax is applied:

\alpha = \text{softmax}(\text{masked scores})

This ensures future tokens receive zero attention probability.

Intuition

For a sequence like “I love AI”:

  • “I” can only attend to “I”

  • “love” can attend to “I”, “love”

  • “AI” can attend to “I”, “love”, “AI”
    No token can access future tokens.

Why It Is Used

Causal masking is used to:

  • Prevent information leakage from future tokens during training

  • Enable autoregressive generation (predicting one token at a time)

  • Ensure training matches inference conditions where future tokens are not available

Where It Is Not Used

It is not used in encoder-only models like BERT, where attention is bidirectional and every token can attend to all others.

Key Idea

Causal masking enforces a strict left-to-right dependency so the model behaves like a real text generator that cannot “peek” into the future.

Interview-Ready Answer

Causal masking is a technique used in Transformer self-attention to prevent tokens from attending to future positions in a sequence. It is applied by masking the upper triangular part of the attention score matrix before softmax, setting those values to negative infinity so their contribution becomes zero. This ensures each token only depends on past tokens, preventing information leakage and enabling autoregressive text generation in models like GPT.

Why does setting masked positions to −∞ (not 0) work correctly after softmax?

Why setting masked positions to −∞ (not 0) works correctly after softmax

Definition

In attention mechanisms like Transformers, masking is used to prevent the model from attending to certain positions. This is done by adding a mask before applying softmax. Masked positions are assigned a very large negative value (−∞ in theory) so that they become irrelevant after softmax.

Why it is used

It ensures that:

  • The model completely ignores unwanted positions

  • Probability distribution remains valid

  • No information leaks from future tokens or padding tokens

How it works step-by-step

Suppose attention scores are [2, 1, 0] and the second position is masked

Masked scores become [2, −∞, 0]

Softmax formula is
\text{softmax}(x_i)=\frac{e^{x_i}}{\sum_j e^{x_j}}

Step 1: Apply exponentiation

  • e² = large value

  • e⁰ = 1

  • e^(−∞) = 0

Step 2: Result becomes [e², 0, 1]

Step 3: Normalize

  • Masked position contributes 0

  • Final probability for masked token becomes 0

  • Remaining probabilities are redistributed among valid positions

Key components

  • Attention logits

  • Mask (0 for valid, −∞ for invalid)

  • Softmax function

  • Exponential normalization

Why not use 0 instead of −∞

If we use 0:

  • Scores like [2, 0, 0] still produce non-zero probabilities for masked positions

  • Masked tokens would still influence output

  • 0 is treated as a valid score, not “ignore this”

Advantages

  • Ensures masked positions get exactly zero probability

  • Prevents information leakage

  • Works naturally with softmax normalization

  • Simple and widely used in Transformers

Disadvantages

  • −∞ is approximated as a large negative number in practice (like −1e9)

  • Can cause numerical stability issues if not handled carefully

Simple real-world example

When predicting the next word in a sentence, a language model should not look at future words. So future token positions are masked with −∞ so their contribution becomes zero after softmax, ensuring the model only learns from valid past context.

Short interview-ready answer

Setting masked positions to −∞ ensures that after applying softmax, their exponent becomes zero, resulting in zero probability. This effectively removes them from attention. Using 0 would still give them non-zero probability, which would break the masking behavior. Hence −∞ correctly enforces exclusion in attention mechanisms.

What is positional encoding, and why can a Transformer not function without position info?

Definition

Positional encoding is a technique used in Transformers to inject information about the order of tokens in a sequence. Since self-attention treats tokens as a set (not a sequence), positional encoding tells the model where each token appears in the sequence.

Why it is used

Transformers do not inherently understand word order. Without positional information, the sentence “dog bites man” and “man bites dog” would look the same to the model because attention alone only sees relationships, not positions.

So positional encoding is used to:

  • Preserve word order

  • Help model understand sequence structure

  • Enable learning of syntax and meaning based on position

How it works step-by-step

Step 1: Token embeddings are created for each word
Each token gets a vector representation

Step 2: Positional encoding is generated
Each position index (0, 1, 2, …) is mapped to a vector using either:

  • Fixed sinusoidal functions

  • Or learned embeddings

Step 3: Combine embeddings
Token embedding + positional encoding are added together

Step 4: Feed into Transformer
The model now sees both meaning (token) and position (order)

Key components

Token embeddings
Position embeddings or sinusoidal encodings
Element-wise addition
Self-attention layers

Why a Transformer cannot work without it

Self-attention alone is permutation-invariant, meaning:

  • It does not care about order

  • It only sees relationships between tokens

So without positional encoding:

  • “I love cats” = “cats love I” (same representation)

  • The model loses sentence structure

  • Language meaning becomes ambiguous

Types of positional encoding

  1. Fixed (sinusoidal)
    Uses sine and cosine functions to encode positions
    Helps generalize to longer sequences

  2. Learned positional embeddings
    Each position has a trainable vector
    Learns best representation from data

Advantages

  • Adds order information to attention-based models

  • Enables parallel processing like Transformers

  • Works well with long sequences

Disadvantages

  • Fixed encoding may not adapt perfectly to all tasks

  • Learned encoding may not generalize to unseen sequence lengths (in some cases)

Simple real-world example

Think of a book where all words are written on separate cards and shuffled. Self-attention can relate words but cannot know order. Positional encoding is like numbering each card so the model knows the correct sequence of the story.

Short interview-ready answer

Positional encoding is used to inject information about the order of tokens in a sequence because self-attention alone is permutation-invariant and does not capture word order. Without it, a Transformer cannot distinguish between different sequences with the same words in different arrangements. Positional encodings are added to token embeddings so the model can learn both meaning and position information.

Explain RoPE and where in the attention pipeline it is applied?

Definition

Rotary Positional Embeddings (RoPE) is a method to inject position information into Transformers by rotating the query and key vectors based on token positions. Instead of adding positional vectors to embeddings, RoPE encodes position directly into the attention computation through rotations.

Why RoPE is used

Traditional positional encodings are added to embeddings, but RoPE:

  • Encodes relative position more naturally

  • Works directly inside attention

  • Preserves dot-product structure

  • Improves extrapolation to longer sequences

Core idea

RoPE applies a position-dependent rotation to each pair of dimensions in query and key vectors. The rotation angle depends on token position.

So instead of modifying embeddings, it modifies:

  • Query (Q)

  • Key (K)

Where RoPE is applied in the attention pipeline

RoPE is applied after Q and K are computed, but before attention scores are calculated.

Full pipeline:

Step 1: Input embeddings
Tokens are converted into vectors

Step 2: Linear projections
Compute:

  • Q = XWq

  • K = XWk

  • V = XWv

Step 3: Apply RoPE (CRITICAL STEP)
Apply rotation to Q and K based on position index
Important: only Q and K are modified, not V

Step 4: Compute attention scores
\text{Attention}(Q,K,V)=\text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)V

Step 5: Continue normal attention computation
Softmax → weighted sum of V

How RoPE works (intuitively)

RoPE treats each pair of dimensions in Q and K as a 2D vector and rotates it by an angle proportional to position.

For position p:

  • Q becomes Q_rot(p)

  • K becomes K_rot(p)

This means:

  • Positions closer together have similar rotation alignment

  • Dot product naturally encodes relative distance

Key components

  • Query vectors (Q)

  • Key vectors (K)

  • Position-dependent rotation matrices

  • Sinusoidal functions (cos, sin)

  • Dot product attention

Why it works

When Q and K are rotated by position:

  • The dot product depends on relative positions (i − j)

  • This gives the model built-in awareness of token distance

  • No explicit positional addition is needed

Advantages

  • Encodes relative position naturally

  • Better long-sequence generalization

  • No need for separate positional embeddings

  • Efficient and mathematically clean

Disadvantages

  • More complex than additive positional encoding

  • Harder to interpret intuitively

  • Requires careful implementation in practice

Simple real-world example

Imagine two people spinning compasses before comparing directions. The angle of each compass depends on where they are standing. When you compare them, you don’t just compare direction, but also how far apart they are. RoPE works similarly: position is encoded as rotation, and attention compares rotated vectors.


Short interview-ready answer

RoPE injects position information by applying a position-dependent rotation to query and key vectors before the attention score computation. It is applied after linear projections (Q and K are computed) and before the dot-product attention step. This rotation encodes positional information into the vectors so that their dot product reflects relative positions, allowing the model to capture sequence order without adding explicit positional embeddings.

Contrast ALiBi vs RoPE vs learned positional embeddings—what trade-offs do they represent?

Definition

All three methods—ALiBi, RoPE, and learned positional embeddings—inject position information into Transformers, but they do it in fundamentally different ways and reflect different design priorities: simplicity, generalization, and flexibility.

Learned positional embeddings

How it works

Each position index (0, 1, 2, …) has a trainable vector added to token embeddings before attention.

So input becomes:
token embedding + learned position vector

Where it is applied

Before Q, K, V projections (at embedding level)

Trade-offs

Advantages

  • Very flexible (model learns best positional representation)

  • Simple to implement

  • Works well within trained sequence length

Disadvantages

  • Poor extrapolation to longer sequences (out-of-distribution positions)

  • Position meaning is not structured (just learned lookup table)

  • Cannot naturally generalize beyond training context window

RoPE (Rotary Positional Embeddings)

How it works

Instead of adding position, RoPE rotates Q and K vectors based on position index using sinusoidal rotations.

Position is embedded into the geometry of attention.

Where it is applied

After Q and K are computed, before attention scores:

  • Q → rotated Q

  • K → rotated K

  • V unchanged

Then attention is computed normally.

Trade-offs

Advantages

  • Encodes relative position naturally (distance-based reasoning)

  • Strong performance in long-context models

  • Better extrapolation than learned embeddings

  • No extra learned parameters

Disadvantages

  • More mathematically complex

  • Harder to modify or interpret

  • Requires careful implementation for scaling/context extension

ALiBi (Attention with Linear Biases)

How it works

Instead of modifying embeddings or vectors, ALiBi adds a position-dependent bias directly to attention scores:

  • Farther tokens get a stronger negative penalty

  • Bias grows linearly with distance

So attention becomes:
score = QKᵀ + bias(distance)

Where it is applied

After computing QKᵀ, just before softmax

Trade-offs

Advantages

  • Extremely simple (no embeddings or rotations)

  • Very efficient

  • Strong extrapolation to longer sequences

  • No learned positional parameters required

Disadvantages

  • Less expressive than RoPE or learned embeddings

  • Only encodes distance linearly (not rich structure)

  • May underperform on tasks needing complex positional reasoning

Key conceptual comparison

Learned positional embeddings

Position is a learned lookup table
Trade-off: high flexibility, poor extrapolation

RoPE

Position is encoded via geometric rotation in Q/K space
Trade-off: strong structure, good extrapolation, moderate complexity

ALiBi

Position is a linear penalty added to attention scores
Trade-off: extreme simplicity, excellent extrapolation, lower expressiveness

Where each modifies the pipeline

  • Learned embeddings: before attention (at input level)

  • RoPE: inside Q and K before dot product

  • ALiBi: after dot product, before softmax

Simple real-world analogy

  • Learned embeddings: memorizing position labels like “seat 1, seat 2”

  • RoPE: encoding position as orientation (like rotating a compass based on location)

  • ALiBi: applying a distance-based penalty (like preferring nearby seats over far ones)

Interview-ready answer

Learned positional embeddings add trainable position vectors to token embeddings, offering flexibility but poor extrapolation beyond training lengths. RoPE encodes position by rotating query and key vectors, embedding relative position directly into attention and enabling strong long-context generalization. ALiBi instead adds a linear distance-based bias to attention scores before softmax, making it extremely simple and efficient with strong extrapolation, but less expressive than RoPE. Each represents a trade-off between flexibility (learned), structural inductive bias (RoPE), and simplicity with scalability (ALiBi).

When training with packed documents, what issues do absolute positional embeddings introduce?

Definition

Packed documents training means multiple independent text sequences (documents) are concatenated into a single long sequence to improve GPU efficiency, while still masking attention so each document does not attend to others.

Absolute positional embeddings assign each token a fixed position index like 0, 1, 2, 3 regardless of which document it belongs to.

Core issue

Absolute positional embeddings assume position = meaning, but packed documents break that assumption because position now depends on packing, not document structure.

What goes wrong step-by-step

Step 1: Packing documents

Example:

  • Doc A: “The cat sat”

  • Doc B: “Transformers are great”

Packed sequence:
“The cat sat Transformers are great”

Now positions become:

  • Doc A tokens: positions 0,1,2

  • Doc B tokens: positions 3,4,5

But these positions are artificial, not semantic.

Issue 1: Position becomes meaningless across documents

Absolute embeddings treat position 3 as always “third word in a document,” but in packed training:

  • position 3 could be start of a new document

  • model cannot distinguish boundary shifts

So same position index means different things depending on packing.

Issue 2: Cross-document positional bias leakage

Even though attention masking prevents token interaction, positional embeddings still encode:

  • “this token is late in sequence”

  • even if it is actually the start of a new document

So model learns misleading patterns like:

  • later positions behave differently, regardless of document context

Issue 3: Distribution shift between training and inference

During inference:

  • single document starts at position 0

During packed training:

  • documents often start at random positions (e.g., 300, 1200, etc.)

So model learns:

  • “start of document ≠ position 0 always”

This breaks consistency.

Issue 4: Confusing document boundaries

Even with attention masks, absolute positions do not reset at document boundaries, so the model:

  • cannot infer “new document starts here” from position alone

  • must rely entirely on masking, increasing learning burden

Key components affected

  • Absolute positional embeddings

  • Sequence indexing

  • Attention masking (still correct, but not sufficient)

  • Document boundary handling

Why this is a structural problem

Absolute positional encoding assumes a single continuous sequence with stable semantics of position. Packed training violates this because:

  • multiple sequences share one position axis

  • position no longer corresponds to true linguistic structure

Why relative methods avoid this

Methods like RoPE or ALiBi depend on:

  • relative distances or biases
    not absolute indices

So they remain stable under packing because:

  • only differences or distances matter

  • not global position identity

Simple real-world example

Imagine labeling seats in a theater starting from 1 every show. If you combine multiple shows into one long row and keep numbering continuously, seat “10” no longer means anything consistent across shows. Someone sitting at seat 10 could be at the start of a new show or the middle of another. Absolute positional embeddings suffer the same confusion.

Short interview-ready answer

Absolute positional embeddings create problems in packed document training because they assign fixed position indices that no longer correspond to true document structure. When multiple documents are concatenated, position IDs continue across boundaries, causing the model to learn misleading positional patterns and making position semantics inconsistent between training and inference. This introduces distribution shift and weakens the model’s ability to recognize document boundaries, which is why relative methods like RoPE or ALiBi handle packed sequences more naturally.

How does bidirectional encoder self-attention differ from causal decoder self-attention?

Definition

Self-attention is the mechanism where each token in a sequence attends to other tokens to build context-aware representations. The difference between bidirectional (encoder) and causal (decoder) self-attention lies in what each token is allowed to “see” and why.

Bidirectional self-attention (Encoder)

Mechanics

In encoder self-attention:

  • Each token can attend to all tokens in the sequence

  • No masking is applied (except padding masks)

  • Attention matrix is fully connected

So for a sequence of length n, every token can use information from:

  • Left context

  • Right context

  • Itself

Intent

The goal is understanding the full sequence:

  • Build rich contextual representations

  • Capture global meaning of the sentence

  • Learn relationships between all tokens

This is why encoders are used for classification, embeddings, and understanding tasks.

Causal self-attention (Decoder)

Mechanics

In decoder self-attention:

  • Each token can only attend to itself and previous tokens

  • A causal (triangular) mask blocks future tokens

  • Attention matrix is lower-triangular

So token at position i can only see:

  • Tokens 1 to i

Intent

The goal is generation:

  • Predict next token based only on past context

  • Prevent “cheating” by looking ahead

  • Ensure autoregressive factorization of probability

Key structural difference

Encoder attention: full matrix (no direction constraint)
Decoder attention: masked matrix (strict left-to-right constraint)

Why this difference matters

Bidirectional attention:

  • Focuses on representation learning

  • Uses full context for better understanding

Causal attention:

  • Focuses on probability modeling of sequences

  • Ensures valid step-by-step generation

Mathematical view

Encoder models learn:
P(representation | full sequence)

Decoder models learn:
P(token_t | token_<t)

Simple real-world analogy

Bidirectional attention is like reading a full sentence before answering a question, using every word to understand meaning.
Causal attention is like writing a sentence word by word without seeing the future words you haven’t written yet.

Short interview-ready answer

Bidirectional self-attention in encoders allows each token to attend to all tokens in the sequence, enabling full contextual understanding for representation learning. In contrast, causal self-attention in decoders restricts each token to attend only to previous tokens using a triangular mask, enforcing autoregressive generation. The encoder’s intent is to build rich bidirectional representations, while the decoder’s intent is to model next-token prediction in a left-to-right manner for text generation.

What does "autoregressive" mean in Transformers, and how does it differ in GPT vs BERT?

Definition

Autoregressive in Transformers means the model generates or predicts text one token at a time, where each token depends only on previous tokens. The sequence is modeled as a chain of conditional probabilities.

Core idea

The joint probability of a sequence is broken down as

P(x_1, x_2, ..., x_n)=\prod_{t=1}^{n} P(x_t \mid x_{<t})

This means each token is predicted using only the past context.

GPT (autoregressive model)

GPT is a decoder-only Transformer trained with this autoregressive objective.

How it works

  • Uses causal self-attention so future tokens are masked

  • Predicts the next token at every step

  • Generates text left to right

  • Trained with next-token prediction loss

Key idea

It learns how to continue text naturally, one token at a time.

BERT (non-autoregressive model)

BERT is an encoder-only Transformer and does not use autoregressive training.

How it works

  • Uses bidirectional self-attention, so it sees full context

  • Random tokens are masked in the input

  • Predicts missing tokens using both left and right context

  • Makes predictions in parallel

Key idea

It is designed for understanding text, not generating it sequentially.

Main difference in objective

GPT learns to predict the next token using only past tokens
BERT learns to predict missing tokens using full context

Architectural difference

GPT uses causal masking and works left to right
BERT uses no causal mask and sees the whole sentence at once

Why this matters

Autoregressive models like GPT are good for generation tasks like chat and writing
BERT-style models are good for understanding tasks like classification and question answering

Simple analogy

GPT is like writing a sentence word by word without knowing future words
BERT is like filling blanks in a full sentence where you can see everything except the missing parts

Short interview-ready answer

Autoregressive means the model predicts tokens sequentially using only previous tokens, factorizing the sequence probability into conditional steps. GPT uses this autoregressive objective with causal self-attention, making it a left-to-right generative model. BERT is not autoregressive; it uses bidirectional attention and a masked language modeling objective where it predicts missing tokens using full context, making it better suited for understanding rather than generation.

Explain role of residual connections in a Transformer block and what fails without them?

Definition

Residual (skip) connections in a Transformer add the input of a sublayer (like attention or feed-forward network) directly to its output. So instead of only using the transformed output, the model keeps the original signal and adds the learned change.

Mathematically:
output = input + Sublayer(input)

Role inside a Transformer block

Each Transformer block has two main sublayers:

  • Self-attention

  • Feed-forward network

Each of them is wrapped with a residual connection:

  • x → Attention(x) + x

  • x → FFN(x) + x

This ensures the original representation is always preserved and refined, not replaced.

Why residual connections are used

1. Improve gradient flow

They allow gradients to flow directly through the network during backpropagation, reducing the vanishing gradient problem in deep networks.

2. Stabilize training

Each layer only needs to learn a small “update” to the representation instead of learning everything from scratch.

3. Preserve information

Important features from earlier layers are not lost as depth increases.

What fails without residual connections

1. Training becomes unstable

Deep Transformers become hard to optimize because gradients either vanish or explode.

2. Performance degrades with depth

Adding more layers may not improve performance and can even make it worse.

3. Information loss

Each layer fully replaces the previous representation, causing loss of useful low-level features.

4. Slower convergence

The model struggles to learn meaningful transformations efficiently.

Intuition

Without residuals, every layer is forced to completely rewrite the representation. With residuals, each layer only learns a correction or refinement of the previous representation.

Simple real-world analogy

Think of editing a document:

  • Without residuals: you rewrite the entire document at every step

  • With residuals: you keep the original and only make edits or improvements

Short interview-ready answer

Residual connections in Transformers add the input of each sublayer directly to its output, allowing layers to learn refinements instead of complete transformations. They improve gradient flow, stabilize training, and preserve information across deep networks. Without them, Transformers become difficult to train, suffer from vanishing gradients, lose important features across layers, and often fail to benefit from increased depth.

Describe the feed-forward network in a Transformer block—what it computes and why it exists?

Definition

The feed-forward network (FFN) in a Transformer block is a small neural network applied independently to प्रत्येक token after self-attention. It transforms each token’s representation using two linear layers with a non-linearity in between.

What it actually computes

For each token vector x, the FFN computes:

FFN(x) = W2 * σ(W1 * x + b1) + b2

where:

  • W1 expands the dimension (for example d_model → 4d_model)

  • σ is a non-linear activation like GELU or ReLU

  • W2 projects it back to original dimension

So it does:

  1. Expand features

  2. Apply non-linearity

  3. Compress back

Key intuition

Self-attention mixes information across tokens
FFN processes information within each token independently

So:

  • Attention = communication between tokens

  • FFN = computation inside each token

Why it is there

1. Adds non-linearity

Without FFN, stacked attention layers would behave mostly like linear transformations, limiting model expressiveness.

2. Feature transformation

It refines each token representation by:

  • extracting higher-level features

  • re-encoding information learned from attention

3. Increases model capacity

The expansion layer (often 4x hidden size) allows the model to learn richer intermediate representations.

4. Complements attention

Attention decides what information to gather
FFN decides how to process that information

What would fail without FFN

  • Model becomes too linear and weak in expressiveness

  • Poor ability to learn complex patterns

  • Stacking attention layers alone would not significantly increase power

  • Performance drops heavily on language understanding tasks

Simple analogy

Self-attention is like gathering notes from different people in a meeting
FFN is like thinking deeply about those notes individually and rewriting them in a clearer form

Short interview-ready answer

The feed-forward network in a Transformer is a position-wise two-layer MLP applied independently to each token after self-attention. It expands the token dimension, applies a non-linearity, and projects it back. Its role is to add non-linearity and increase model capacity, allowing each token to be transformed and refined after attending to other tokens. Without the FFN, the model would be largely linear and significantly less expressive, reducing its ability to learn complex patterns.

FFN as “memory bank + generalizer”—what does that mean in Transformers?

Definition

The feed-forward network (FFN) in Transformers is often described as doing two roles at once: storing knowledge (memory bank) and transforming information (generalizer). This comes from how its parameters behave at scale.

What “memory bank” means

The FFN’s weights, especially the first linear layer, act like a large store of learned patterns.

Why it is seen as memory

  • Each neuron can respond strongly to specific patterns in token representations

  • Over training, neurons specialize in detecting features like syntax, facts, or concepts

  • The large expansion dimension (often 4× hidden size) allows many such “feature detectors”

So instead of storing facts explicitly, the FFN encodes them in distributed activations across neurons and weights.

In this sense:

  • The weights = long-term learned knowledge

  • The activations = retrieval of that knowledge for a token

What “generalizer” means

After retrieving stored patterns, the FFN transforms them into useful representations.

Why it generalizes

  • Non-linearity (GELU/ReLU) allows complex feature interactions

  • Second linear layer recombines features into a refined representation

  • Works independently per token, applying learned transformations consistently across contexts

So it doesn’t just recall patterns, it reshapes them for the current context.

How both roles coexist

Step 1: Attention brings in information

Tokens collect relevant context from other tokens.

Step 2: FFN acts like memory lookup

Each token activates relevant learned features inside FFN weights.

Step 3: FFN generalizes

It transforms those features into a higher-level representation useful for the next layer.

So:

  • Attention = fetch from context

  • FFN = fetch from learned knowledge + transform it

Why this interpretation exists

Empirically:

  • FFN parameters contain most of the model’s capacity

  • Many “fact-like” associations are stored in FFN weights

  • Different neurons respond to specific concepts or patterns

This makes FFNs behave like distributed key-value memory systems:

  • Keys = input patterns activating neurons

  • Values = transformed feature outputs

Key insight

Self-attention handles dynamic context
FFN handles static learned knowledge transformation

Together they form:

  • context-based reasoning + parametric knowledge storage

Simple analogy

Attention is like asking other people in a room for information
FFN is like your internal knowledge base and reasoning system that interprets what you heard

Short interview-ready answer

The FFN in a Transformer is often viewed as both a memory bank and a generalizer. It acts as a memory bank because its large weight matrices store distributed patterns and associations learned during training, where specific neurons respond to specific features or concepts. It acts as a generalizer because it applies nonlinear transformations to these activated features, refining and recombining them into useful representations. Together, this allows attention to retrieve context while the FFN injects learned knowledge and performs nonlinear computation on each token independently.

Why has GELU largely replaced ReLU in Transformer FFNs?

Definition

GELU (Gaussian Error Linear Unit) is an activation function used in Transformer feed-forward networks that smoothly gates inputs based on their value, unlike ReLU which hard-clips negative values to zero.

Why activation functions matter in FFNs

In Transformer FFNs, the activation function is the only source of non-linearity inside each layer. It strongly affects:

  • how information flows through neurons

  • how smoothly gradients propagate

  • how well representations are learned

ReLU behavior

ReLU is defined as
ReLU(x) = max(0, x)

What it does:

  • Keeps positive values unchanged

  • Completely removes negative values

  • Creates sparse activations

GELU behavior

GELU can be thought of as:

  • A smooth probabilistic version of ReLU

  • It weights inputs instead of hard-cutting them

\text{GELU}(x)=x\cdot P(X \le x)

Intuition:

  • Small or negative values are not fully removed

  • They are softly downweighted

  • Larger values pass through more strongly

Why GELU replaced ReLU in Transformers

1. Smooth gradients improve optimization

ReLU has a sharp kink at 0, which can cause:

  • abrupt gradient changes

  • dead neurons (always zero output)

GELU is smooth everywhere, making training more stable in deep Transformer stacks.

2. Better information flow

ReLU completely drops negative signals
GELU preserves some negative information in a softened form

This is important because:

  • negative activations can still carry meaningful signal in language representations

  • hard zeroing can discard useful nuance

3. Empirical performance gains

Across large-scale language models, GELU consistently shows:

  • slightly better perplexity

  • improved convergence stability

  • better downstream task performance

Even small improvements matter at scale.

4. Better fit for dense representations

Transformers rely on dense, continuous representations rather than sparse ones.
ReLU introduces sparsity, which is not always desirable in FFNs.

GELU preserves dense information flow.

5. Compatibility with large-scale training

GELU behaves more predictably under:

  • very deep networks

  • large batch training

  • mixed-precision training

This makes it more reliable in modern LLM setups.

Key trade-off

ReLU:

  • simpler

  • faster

  • sparse

GELU:

  • smoother

  • more expressive

  • better performance in practice

Simple analogy

ReLU is like a strict filter that completely blocks weak signals
GELU is like a soft filter that reduces weak signals but still lets some information through

Short interview-ready answer

GELU has largely replaced ReLU in Transformer FFNs because it provides a smooth, differentiable activation that preserves more information and improves gradient flow. Unlike ReLU, which hard-zeros negative values and can lead to dead neurons and loss of signal, GELU softly weights inputs based on their magnitude, allowing better representation learning. This leads to more stable optimization and consistently better performance in large-scale language models, making it the preferred choice in modern Transformers.

What is SwiGLU, and why do modern LLMs prefer it over GELU/ReLU?

Definition

SwiGLU (Swish Gated Linear Unit) is a gated activation used in modern Transformer feed-forward networks where the input is split into two streams, one acts as content and the other acts as a learned gate that controls information flow.

What it computes

Instead of a single activation like GELU or ReLU, SwiGLU does:

FFN(x) = (xW1) ⊙ Swish(xW2) W3
where Swish(x) = x · sigmoid(x)

So:

  • one projection produces values

  • another projection produces gates

  • element-wise multiplication decides what passes through

Key idea

It replaces fixed non-linearity with learned gating, meaning the model decides feature-wise what to keep or suppress for each token.

Why SwiGLU is used in modern LLMs

1. Better expressiveness

Instead of applying the same activation everywhere (like GELU), SwiGLU learns dynamic, input-dependent filtering of features.

2. Feature-wise control

Each dimension can be independently scaled up or down using the gate, allowing more precise representation control.

3. Improved performance at scale

Large models consistently show:

  • lower loss

  • better downstream performance

  • better scaling behavior compared to GELU/ReLU FFNs

4. Parameter efficiency

SwiGLU often achieves better quality without simply increasing model width, making it more compute-efficient.

5. Better gradient flow

The gating mechanism helps gradients flow through important features while suppressing noisy ones, leading to more stable training.

Key difference from GELU/ReLU FFNs

GELU/ReLU FFN:

  • single linear path

  • fixed non-linearity applied uniformly

SwiGLU FFN:

  • two linear paths

  • one for content, one for gating

  • multiplicative interaction between them

Intuition

GELU is like smoothing every signal in the same way
SwiGLU is like dynamically deciding which signals should be amplified or muted for each token

Simple analogy

GELU is a fixed filter applied to all inputs equally
SwiGLU is a mixing board where each channel has its own learned volume control based on context

Short interview-ready answer

SwiGLU is a gated feed-forward activation used in modern Transformers where the input is split into two projections, one acting as content and the other as a learned gate, and they are multiplied element-wise. This replaces fixed activations like GELU with dynamic feature-wise gating, improving expressiveness, training stability, and parameter efficiency. This is why many modern LLMs prefer SwiGLU over traditional FFN activations.

Why is layer norm used in Transformers instead of batch norm? Give a concrete reason?

Definition

Layer Normalization is a technique that normalizes the activations of a single token across its feature dimensions. Batch Normalization instead normalizes activations across the batch dimension.

Key difference

LayerNorm works per token, independent of batch size.
BatchNorm depends on statistics computed across the entire batch.

Why Transformers use LayerNorm instead of BatchNorm

1. Batch size independence

Transformers often use small or variable batch sizes, especially in large models or during inference. BatchNorm relies on batch statistics, so its behavior changes when batch size changes. LayerNorm does not have this problem because it normalizes each token independently.

2. Works with variable sequence lengths

NLP inputs have different lengths and include padding tokens. BatchNorm mixes information across sequences in a batch, which makes padding corrupt the statistics. LayerNorm avoids this because it operates only within each token.

3. Stable training and inference consistency

BatchNorm behaves differently during training (uses batch statistics) and inference (uses running averages). This creates a mismatch. LayerNorm behaves the same in both modes.

4. Compatible with autoregressive models

In models like GPT, tokens are generated one at a time. Batch-level statistics are not meaningful in this setting. LayerNorm works naturally in sequential generation.

What would fail with BatchNorm in Transformers

It would introduce instability due to changing batch statistics, break consistency between training and inference, and be affected by padding and variable sequence lengths.

Simple intuition

BatchNorm normalizes across examples in a batch.
LayerNorm normalizes within each individual token.

Simple analogy

BatchNorm is like adjusting scores by comparing students in a class.
LayerNorm is like grading each student based only on their own performance scale.

Short interview-ready answer

Transformers use LayerNorm instead of BatchNorm because sequence models deal with variable-length inputs, small or changing batch sizes, and autoregressive generation, all of which make batch statistics unstable. BatchNorm depends on batch-wide statistics and behaves differently during training and inference, while LayerNorm normalizes each token independently across features, making it stable, consistent, and well-suited for Transformer architectures.

Why does attention weight matrix have shape (seq_len × seq_len), not embedding size?

Definition

In self-attention, the attention weight matrix is a square matrix that represents how much each token in a sequence attends to every other token in the same sequence. Its shape is always:

[
(\text{seq_len} \times \text{seq_len})
]

It does not depend on embedding dimension, only on the number of tokens.

Why it is used

The attention matrix is used to:

  • Decide which tokens should influence each other

  • Assign importance scores between every pair of tokens

  • Build a context-aware representation of each token

How it works step-by-step

Assume we have a sequence of length n = seq_len.

Step 1: Input embeddings

Each token is represented as a vector:

  • Input shape:
    [
    (n \times d)
    ]
    where (d) = embedding dimension

Step 2: Create Q, K, V

We compute:

  • Queries (Q)

  • Keys (K)

  • Values (V)

Each becomes:
[
(n \times d_k)
]

Step 3: Compute attention scores

We multiply:

[
Q \cdot K^T
]

Shape breakdown:

  • Q: ( (n \times d_k) )

  • K^T: ( (d_k \times n) )

Result:
[
(n \times n)
]

This gives a score between every token pair.

Step 4: Softmax normalization

Apply softmax row-wise:

[
\text{softmax}(QK^T)
]

Still:
[
(n \times n)
]

This is the attention weight matrix.

Step 5: Apply weights to values

Final output:

[
(n \times n) \cdot (n \times d_k) = (n \times d_k)
]

So embedding dimension affects the final output, not the weight matrix.

Key components

  • Seq length (n): defines matrix size

  • Q, K: used to compute similarity

  • Softmax: converts scores into probabilities

  • V: carries actual information

Why shape is always (seq_len × seq_len)

Because attention is fundamentally:

  • “For each token, how much should I attend to every other token?”

So:

  • Rows = query tokens (who is looking)

  • Columns = key tokens (who is being looked at)

That creates a full pairwise comparison:
[
n \times n
]

Embedding dimension only affects:

  • how similarity is computed

  • not how many tokens exist

Advantages

  • Captures relationships between all tokens

  • Works regardless of embedding size

  • Fully parallel computation

  • Handles long-range dependencies well

Disadvantages

  • Quadratic complexity (O(n^2))

  • Memory-heavy for long sequences

  • Not efficient for very large inputs

The attention weight matrix always has shape (seq_len × seq_len) because it represents pairwise interactions between all tokens in the sequence. When we compute attention, we multiply Q and Kᵀ, which transforms the embedding dimension out of the equation and results in a matrix based only on sequence length. Each row corresponds to how one token attends to all others, and each column represents contributions from all tokens. Therefore, embedding dimension affects how similarity is computed but not the final attention matrix shape.

What is the KV cache, and how does it speed up autoregressive decoding?

Definition

KV cache (Key-Value cache) is an optimization used in transformer models during autoregressive decoding, where previously computed Keys (K) and Values (V) from earlier tokens are stored so they don’t need to be recomputed at every generation step.

It is mainly used during inference (text generation), not training.

Why it is used

Without KV cache:

  • At every new token generation, the model recomputes attention for all previous tokens again and again

  • This becomes very slow: repeated computation grows with sequence length

With KV cache:

  • Previously computed K and V are stored and reused

  • Only the new token’s Q, K, V are computed each step

This dramatically improves speed.

How it works step-by-step

Assume we are generating text token by token.

Step 1: First token

  • Input: token₁

  • Compute:

    • Q₁, K₁, V₁

  • Store:

    • KV cache = {K₁, V₁}

Step 2: Second token

  • Input: token₂

  • Compute only:

    • Q₂, K₂, V₂

  • Retrieve cached:

    • K₁, V₁

  • Attention uses:

    • Q₂ with [K₁, K₂]

    • Values [V₁, V₂]

  • Update cache:

    • KV cache = {K₁, K₂, V₁, V₂}

Step 3: Third token

  • Compute only new:

    • Q₃, K₃, V₃

  • Reuse cached K₁, K₂, V₁, V₂

  • Attention is computed over all past tokens efficiently

Key idea

Instead of recomputing past transformations:

We change from:

Recompute everything each step → store and reuse past computations

What exactly is stored?

For each transformer layer:

  • Keys (K) for all previous tokens

  • Values (V) for all previous tokens

Not stored:

  • Queries (Q), because they are only needed for the current token

Why it accelerates decoding

Without KV cache:

For step t:

  • Attention cost ≈ O(t²) per step (recomputing all past interactions repeatedly)

Total cost becomes very large.

With KV cache:

For step t:

  • Only compute:

    • Q for current token

    • K and V for current token

  • Attention uses cached K, V

Cost per step becomes:

  • O(t) instead of O(t²)

So generation becomes much faster and scalable.

Key components

  • Query (Q): current token’s attention request

  • Key (K): stored representations of past tokens

  • Value (V): stored content vectors of past tokens

  • Cache: memory storing all past K and V tensors

Advantages

  • Huge speedup in autoregressive decoding

  • Avoids redundant computation

  • Enables real-time text generation

  • Scales better for long sequences

Disadvantages

  • Uses more GPU memory (stores all past K/V)

  • Cache grows with sequence length

  • Not useful during training (only inference)

  • Harder to manage in very long contexts

Simple real-world analogy

Imagine reading a book:

Without KV cache:

  • Every time you read a new page, you re-read and re-analyze all previous pages

With KV cache:

  • You keep notes (K and V) from previous pages

  • When reading a new page, you only process it once and refer to your notes

Short interview-ready answer

KV cache is an optimization used in transformer inference where the Keys and Values from previous tokens are stored during autoregressive decoding. Instead of recomputing attention over all past tokens at every step, the model reuses cached K and V and only computes them for the new token. This reduces redundant computation, making decoding much faster and reducing per-step complexity from quadratic to linear with respect to sequence length, at the cost of additional memory usage.

Differentiate prefill vs decode phase in LLM inference and what dominates cost in each?

Definition

During LLM inference, text generation is split into two main phases:

  • Prefill phase: processing the input prompt all at once

  • Decode phase: generating output tokens one by one autoregressively

Both use transformer attention, but their computation patterns are very different.

1. Prefill Phase

What it is

The prefill phase processes the entire input prompt in one forward pass before generation begins.

Example:
Input prompt:
“I want to book a flight from Delhi to Mumbai”

The model processes all tokens together.

How it works

  • Input tokens are passed through all transformer layers at once

  • Self-attention is computed over the full sequence

  • KV cache is initialized and filled for all tokens

Computation pattern

For sequence length (n):

  • Attention computed over full matrix:
    [
    (n \times n)
    ]

So each token attends to all others in the prompt.

What dominates cost

Main cost driver:

  • Full self-attention over the entire prompt

Why:

  • Quadratic complexity (O(n^2))

  • Large matrix multiplications across all layers

So:

  • Prefill is compute-heavy due to attention over long context

2. Decode Phase

What it is

The decode phase generates output tokens one at a time.

Example:
After prompt → model generates:

  • Token 1

  • Token 2

  • Token 3
    ... sequentially

How it works

For each new token:

  • Compute Q, K, V only for the new token

  • Use KV cache for all previous tokens

  • Perform attention between:

    • current token (query)

    • all past tokens (cached keys/values)

Computation pattern

At step (t):

  • Attention is:
    [
    (1 \times t)
    ]

So each new token attends over all previous tokens.

What dominates cost

Main cost driver:

  • Memory bandwidth + KV cache attention lookup

Why:

  • Computation per step is linear (O(t)), not quadratic

  • But cost grows over time due to longer context

  • Bottleneck becomes:

    • reading cached K/V from memory

    • not recomputing math

So:

  • Decode is memory-bound rather than compute-bound

Short interview-ready answer

The prefill phase processes the entire input prompt in a single forward pass and computes full self-attention across all tokens, making it compute-heavy with quadratic complexity. As a result, decode is linear per token and becomes memory-bandwidth bound due to repeated KV cache reads, while prefill is dominated by compute-intensive full attention over the prompt.

What happens if output exceeds the model’s context window, functionally and in quality?

Definition

The context window of an LLM is the maximum number of tokens the model can attend to at once (input + generated output). If generation tries to go beyond this limit, older tokens must be dropped or cannot be attended to anymore.

What happens functionally

1. Sliding / truncation of context

Most systems handle overflow in one of these ways:

  • Hard truncation

    • Oldest tokens are removed from memory

    • KV cache for those tokens is discarded

  • Sliding window (common in serving systems)

    • Only the most recent N tokens are kept

    • Attention is restricted to this window

So the model effectively sees:

[
\text{only last } n \text{ tokens of history}
]

2. KV cache limitation

Since KV cache stores past keys and values:

  • Once tokens fall outside the context window:

    • Their K/V entries are dropped

    • They no longer participate in attention

3. Loss of full history awareness

The model can no longer attend to:

  • Early parts of the prompt

  • Earlier generated reasoning

  • Long-range dependencies

It behaves as if those tokens never existed.

What happens to generation quality

1. Loss of coherence

As older context disappears:

  • The model may contradict itself

  • It may "forget" instructions from earlier

  • It may shift tone or style

2. Degraded long-range reasoning

Tasks affected:

  • Math proofs

  • Multi-step reasoning

  • Story consistency

  • Code dependencies

Because earlier steps are no longer visible.

3. Repetition and drift

Common behaviors:

  • Repeating earlier ideas

  • Going off-topic

  • Gradual semantic drift (“topic wandering”)

4. Instruction forgetting

If system or user instructions were early in the prompt:

  • They may be dropped

  • Model may stop following constraints

5. Apparent “memory reset”

To the model, the sequence becomes:

“I only see recent tokens, so I must infer the rest”

This can cause inconsistent outputs.

Key intuition

The model does not “know” it exceeded context.
It simply:

Stops receiving earlier tokens as input.

So it behaves like a person who forgot the beginning of a conversation.

Why this happens

Transformers:

  • Only attend over tokens inside the context window

  • Have no external memory beyond KV cache

  • Cannot reconstruct dropped tokens

So once tokens are out of scope, they are irrecoverable.

Real-world analogy

Imagine writing a long essay on a whiteboard:

  • You can only see the last few lines

  • Earlier parts are erased as you write more

Result:

  • You may contradict earlier arguments

  • You may repeat points

  • You lose overall structure

Mitigations used in practice

  • Summarizing old context into shorter form

  • Sliding window attention

  • Retrieval-Augmented Generation (RAG)

  • External memory systems

Short interview-ready answer

When an output sequence exceeds the model’s context window, older tokens are no longer accessible to attention and are dropped from the KV cache or excluded via a sliding window. Functionally, the model can only condition on the most recent tokens. This leads to loss of long-range dependencies and can cause degraded generation quality such as inconsistency, repetition, topic drift, and forgetting of earlier instructions. Essentially, the model behaves as if earlier parts of the conversation no longer exist, since it cannot attend to or retrieve them.

Explain Grouped-Query Attention (GQA) vs Multi-Head Attention—memory and quality differences?

Definition

Grouped-Query Attention (GQA) is an attention mechanism where multiple query heads share a smaller number of key and value heads.

It sits between:

  • Multi-Head Attention (MHA) → one K/V per head (high cost)

  • Multi-Query Attention (MQA) → one shared K/V for all heads (lowest cost)

So GQA is a middle ground optimization used in modern LLMs.

Vanilla Multi-Head Attention (MHA)

How it works

In standard MHA:

  • Each attention head has its own:

    • Query (Q)

    • Key (K)

    • Value (V)

If there are:

  • (H) heads → we have:

    • (H) different K matrices

    • (H) different V matrices

Each head computes attention independently.

Memory behavior

During inference (KV cache):

  • Store K and V for every head

  • Memory scales as:

[
O(H \cdot n \cdot d)
]

where:

  • (H) = number of heads

  • (n) = sequence length

So memory is large and grows with heads

Grouped-Query Attention (GQA)

How it works

In GQA:

  • Query heads are still many (like MHA)

  • But K and V heads are shared across groups of Q heads

Example:

  • 8 query heads

  • 2 key/value heads

So:

  • Each KV head is reused by 4 query heads

Structure

Instead of:

  • Q₁ K₁ V₁

  • Q₂ K₂ V₂

  • Q₃ K₃ V₃

We have:

  • Q₁, Q₂ → K₁ V₁

  • Q₃, Q₄ → K₂ V₂

  • etc.

Memory impact

KV cache size

In GQA:

  • Only G KV heads instead of H

So memory becomes:

[
O(G \cdot n \cdot d)
]

where:

  • (G \ll H)

Result:

  • Significant reduction in KV cache memory

  • Lower bandwidth during decoding

  • Faster inference

Quality impact

Compared to MHA

Pros of GQA:

  • Very close quality to MHA in practice

  • Retains multiple query perspectives

  • Works well in large models (used in LLaMA-style models)

Cons:

  • Slight reduction in expressiveness of attention keys/values

  • Some fine-grained distinctions across heads are lost

Why quality drop is small

Even though K/V are shared:

  • Queries remain independent

  • Different heads still “look at” shared memory differently

  • Most representational power still comes from Q projections and depth

So:

GQA preserves diversity in what is asked, even if memory is shared

The trade-off is a slight reduction in expressiveness, but in practice the quality degradation is minimal, which is why GQA is widely used in modern large language models.

Describe Flash Attention at a high level—why is it faster than naive attention?

Definition

Flash Attention is a highly optimized algorithm for computing attention in transformers that is designed to be memory-efficient and fast by minimizing data movement between GPU memory (HBM) and on-chip cache (SRAM).

It produces the exact same output as standard attention, but computes it in a much more hardware-efficient way.

Why naive attention is slow

Standard attention computes:

[
\text{Attention}(Q,K,V) = \text{softmax}(QK^T)V
]

Naive implementation problem:

It explicitly forms:

  • The full (QK^T) matrix (size (n \times n))

  • The full attention matrix after softmax

Bottleneck:

  • This matrix is huge

  • It must be written to and read from GPU memory repeatedly

So the real bottleneck is:

Memory bandwidth, not compute

Key idea behind Flash Attention

Instead of materializing the full attention matrix, Flash Attention:

Computes attention in small blocks that fit in fast on-chip memory (SRAM), and never writes the full (n \times n) matrix to memory.

How it works (high level steps)

Step 1: Tiling (blocking)

  • Split Q, K, V into small blocks

  • Each block fits into fast GPU SRAM

Step 2: Blockwise computation

For each Q block:

  • Load a small block of K and V

  • Compute partial attention scores

  • Immediately apply softmax incrementally

Step 3: Online softmax trick

Instead of computing full softmax at once:

  • Maintain running max and normalization

  • Update softmax values incrementally per block

This avoids storing full score matrix.

Step 4: Streaming output

  • Multiply softmax weights with V block-by-block

  • Accumulate final output directly

Explain ring attention and how it enables very long contexts across multiple devices?

Definition

Ring Attention is a distributed attention algorithm that enables transformers to process very long sequences by splitting the attention computation across multiple devices (GPUs/TPUs) in a circular (ring-like) communication pattern.

It is designed to scale attention beyond a single device’s memory limits while still computing exact attention (not an approximation).

Why it is needed

Standard attention has two major problems for long context:

  • Memory grows as (O(n^2))

  • KV cache grows linearly with sequence length

  • A single GPU cannot hold very long sequences

So even with optimizations like Flash Attention, a single device eventually runs out of memory.

Ring Attention solves this by:

distributing sequence chunks across multiple devices and coordinating computation.

Core idea

Instead of putting the full sequence on one device:

  • Split the sequence across multiple GPUs

  • Each GPU holds only a chunk of tokens

  • GPUs pass information to each other in a ring communication pattern

  • Each GPU incrementally computes attention over the full sequence

How it works step-by-step

Assume:

  • Sequence is split across (N) GPUs

  • Each GPU holds a chunk of tokens

Step 1: Partitioning

Each device stores:

  • A slice of Q, K, V tensors

Example:

  • GPU 1 → tokens 1–1000

  • GPU 2 → tokens 1001–2000

  • etc.

Step 2: Local computation

Each GPU computes attention for its local Q against local K/V.

But this is incomplete — it only sees part of the sequence.

Step 3: Ring communication

Now the key idea:

  • Each GPU sends its K/V block to the next GPU in a ring

  • Receives a new block from the previous GPU

  • Repeats this process multiple rounds

So over time:

every GPU sees every other GPU’s tokens, but not all at once

Step 4: Streaming attention accumulation

Each GPU:

  • Computes partial attention scores per received block

  • Uses online softmax accumulation

  • Aggregates results incrementally

No GPU ever stores full (n \times n) matrix.

Step 5: Final result

After completing all ring rotations:

  • Each GPU has computed correct attention for its assigned query tokens

  • Outputs are combined across devices

Key idea

Ring Attention works because:

Attention is associative enough to be computed in chunks and merged incrementally.

So instead of:

  • “all tokens at once”

We do:

  • “tokens in waves across devices”

Why it enables very long context windows

1. Memory scaling across devices

Each GPU only stores:

  • (1/N) of the sequence

So total effective memory becomes:

[
O(n^2 / N)
]

distributed across hardware.

2. KV cache distribution

KV cache is also split:

  • No single device stores full cache

  • Each device stores only its chunk

3. Compute parallelism

Each GPU:

  • works independently on its chunk

  • contributes to global attention collaboratively

Performance trade-offs

Pros

  • Enables extremely long context (hundreds of thousands or more tokens)

  • Exact attention (not approximate like sparse methods)

  • Scales linearly with number of devices

  • Works with standard transformer architecture

Cons

  • Requires frequent inter-device communication

  • Communication overhead can be significant

  • Performance depends heavily on network bandwidth

  • Ring-based communication topology

  • Streaming / online softmax

  • Distributed KV cache

  • Partial attention accumulation

Describe MoE and how Mixtral routes tokens at inference time?

Definition

Mixture of Experts (MoE) is a transformer architecture where instead of using one dense feedforward network for every token, the model contains many specialized “expert” sub-networks, and a router (gating network) selects only a few experts per token.

So instead of:

  • one big model processing everything

MoE uses:

  • many smaller expert models

  • but only activates a subset per input token

Why it is used

MoE is designed to:

  • increase model capacity massively

  • without increasing computation proportionally

Key idea:

Scale parameters up, but keep compute per token low

This allows models like Mixtral to have very large total parameters while still being fast at inference.

High-level architecture

An MoE layer replaces the standard feedforward network (FFN) in a transformer with:

  • Multiple experts (FFNs)

  • A router/gating network

  • A top-k selection mechanism

How routing works (core idea)

For each token:

  1. The token embedding is passed into the router

  2. Router outputs a score for each expert

  3. Top-k experts are selected (usually k = 1 or 2)

  4. Token is sent only to those experts

  5. Outputs are combined (weighted sum)

Step-by-step inference in Mixtral-like models

Let’s walk through what happens for one token.

Step 1: Token enters MoE layer

The hidden state vector (x) is passed into the router.

Step 2: Router computes scores

A small linear layer produces logits:

  • Score for Expert 1

  • Score for Expert 2

  • ...

  • Score for Expert N

Then:

  • Apply softmax (or similar normalization)

Step 3: Top-k selection

Instead of using all experts:

  • Select top-k highest scoring experts

In models like Mixtral:

  • typically k = 2 experts per token

So each token only activates 2 experts out of many (e.g., 8 or more).

Step 4: Dispatching tokens

The token is sent only to selected experts:

  • Expert A processes the token

  • Expert B processes the token

Other experts are idle for that token

Step 5: Expert computation

Each expert is just a feedforward network:

  • independently transforms the token

  • specializes in certain patterns (syntax, reasoning, code, etc.)

Step 6: Combining outputs

Outputs from selected experts are:

  • weighted by router scores

  • summed together

Final result is passed to the next transformer layer.

Key intuition

Instead of:

“Every neuron processes every token”

MoE does:

“Only the most relevant subnetworks process each token”

Why this improves efficiency

1. Parameter scaling without compute explosion

  • Total parameters increase (many experts)

  • But only 1–2 experts are used per token

So compute stays roughly constant.

2. Specialization

Experts naturally specialize:

  • one may handle code-like patterns

  • another may handle factual text

  • another may handle reasoning patterns

3. Better capacity per FLOP

Compared to dense models:

  • more knowledge capacity

  • similar inference cost

What makes Mixtral special

Models like Mixtral (by Mistral AI) use:

  • Sparse MoE layers inside transformers

  • Typically 8 experts

  • Top-2 routing per token

So:

  • Only 2/8 experts activate per token

  • But model still benefits from full 8-expert capacity

Key components

  • Experts: independent FFNs

  • Router (gating network): decides which experts to use

  • Top-k selection: limits active experts

  • Load balancing: prevents overuse of certain experts

Advantages

  • Very large model capacity

  • Efficient inference cost

  • Better specialization of knowledge

  • Scales well to huge parameter counts

Disadvantages

  • Routing complexity

  • Load balancing challenges

  • Less hardware efficiency than dense models in some setups

  • Training instability if routing is poorly balanced

Real-world analogy

Imagine a hospital:

  • Dense model: every doctor treats every patient

  • MoE model: a triage system routes each patient to the 2 most relevant specialists

So:

  • not all experts work on every case

  • only the most relevant ones are activated

Short interview-ready answer

Mixture of Experts (MoE) is a transformer architecture where multiple expert feedforward networks exist, but only a small subset is activated per token using a learned routing mechanism. During inference in models like Mixtral, each token is passed through a router that assigns scores to all experts, selects the top-k (typically 2), and sends the token only to those experts. Their outputs are then combined using router weights. This allows the model to scale to a very large number of parameters while keeping inference cost low, since only a fraction of experts are active for each token.

How are head dim, embedding dim, and number of heads related in multi-head attention?

In multi-head attention, the model splits the full embedding space into multiple smaller “heads”, where each head attends independently to the input.

  • Embedding dimension: (d_{model})

  • Number of heads: (h)

  • Head dimension: (d_{head})

Core relationship (constraint)

The fundamental constraint is:

[
d_{model} = h \times d_{head}
]

or equivalently:

[
d_{head} = \frac{d_{model}}{h}
]

Why this constraint exists

Step 1: Input representation

Each token is represented as a vector of size:

[
(d_{model})
]

Step 2: Splitting into heads

This vector is projected into Q, K, V, and then split into (h) heads.

Instead of one large attention space, we get:

  • (h) parallel attention heads

  • each working on a smaller subspace

So each head gets:
[
d_{head}
]

Step 3: Parallel attention computation

Each head independently computes:

  • Q, K, V of size (d_{head})

  • attention over tokens

Step 4: Concatenation

Outputs from all heads are concatenated:

[
h \times d_{head} = d_{model}
]

Then projected back to model space.

Key intuition

The model:

splits a large representation space into multiple smaller attention subspaces, processes them independently, then recombines them.

Why this design is used

1. Parallel representation learning

Each head can learn different relationships:

  • syntax

  • long-range dependencies

Splitting ensures:

  • total compute stays balanced

3. Expressivity

Multiple heads allow:

different “views” of the same token sequence

  • (d_{model} = 512)

Then:

[

]

So:

  • 8 heads

  • each with 64-dimensional attention space

Key takeaway

The design constraint ensures that:

splitting into multiple heads does not change total representation size, only how it is factorized.

This design allows the model to learn multiple types of relationships in parallel while keeping the overall computational and representational size consistent.

  • Each head output is still separate

learning interactions between different attention heads

Without it, heads remain independent even after merging.

Each head output has size:

[
d_{head}

h \times d_{head} = d_{model}

(W_o):

  • re-projects it into the model’s working space

Describe how causal mask is combined with attention scores and why order matters?

Definition

A causal mask in transformer attention is a constraint that prevents each token from attending to future tokens. It enforces the autoregressive property:

A token can only depend on previous tokens, not future ones.

Where it is applied

In self-attention, we first compute raw attention scores:

[
S = QK^T
]

Then we apply:

  1. causal mask

  2. softmax normalization

So the pipeline is:

[
QK^T \rightarrow \text{mask} \rightarrow \text{softmax}
]

How the causal mask is combined with scores

The mask is applied additively before softmax.

Step 1: Compute raw scores

[
S_{ij} = q_i \cdot k_j
]

Each token (i) can potentially attend to all tokens (j).

Step 2: Apply causal mask

We define a mask matrix (M):

  • (M_{ij} = 0) if (j \le i) (allowed)

  • (M_{ij} = -\infty) if (j > i) (future tokens blocked)

Then:

[
S' = S + M
]

So invalid positions become very large negative values.

Step 3: Apply softmax

[
A = \text{softmax}(S')
]

Because:

  • (\exp(-\infty) = 0)

Future tokens get zero probability.

Why this ordering matters

1. Softmax must see masked scores

Softmax is sensitive to relative magnitudes, so masking must happen before normalization.

If you apply softmax first:

  • probabilities are already distributed over future tokens

  • masking afterward would not remove their influence correctly

2. Ensures true probability distribution

Applying mask before softmax guarantees:

  • probabilities sum to 1 only over valid (past) tokens

  • invalid tokens contribute zero probability

3. Prevents information leakage

If masking is not applied before softmax:

  • future tokens could influence normalization

  • model could indirectly “peek” at future information

That would break autoregressive training.

4. Numerical stability

Using (-\infty) (or large negative number) ensures:

  • masked logits vanish after exponentiation

  • clean and stable computation

Key intuition

Causal masking is like:

putting a wall over future positions before the model decides where to pay attention.

If you apply softmax first, the model has already “looked” at everything.

Real-world analogy

Imagine reading a book while predicting the next word:

  • Raw scores = considering all words in the sentence

  • Mask = covering future words with opaque paper

  • Softmax = deciding what to focus on among visible words

Order matters because:

you must cover the future before making attention decisions

Short interview-ready answer

The causal mask is added to the raw attention logits (QK^T) before applying softmax by setting future token positions to (-\infty), ensuring their attention probabilities become zero. This ordering is critical because softmax normalizes over all logits; applying the mask after softmax would not prevent information leakage from future tokens. By masking first and then applying softmax, the model guarantees a valid probability distribution over only past tokens and preserves the autoregressive property required for language generation.

What is a "context vector" in self-attention, and why is it the key output?

Definition

In self-attention, a context vector is the final output representation for each token after it has aggregated information from all other relevant tokens in the sequence.

For each token (i), the context vector is a weighted combination of value vectors:

[
c_i = \sum_{j=1}^{n} \alpha_{ij} v_j
]

Where:

  • (\alpha_{ij}) = attention weight from token (i) to token (j)

  • (v_j) = value vector of token (j)

A context vector is:

A token’s updated representation that includes information from the entire sequence (or allowed context window)

So instead of being just the original embedding of a token, it becomes:

  • context-aware

  • relationship-enriched

  • task-relevant

Step 1: Compute attention scores

[
S = QK^T
]

Step 2: Normalize with softmax

[
\alpha_{ij} = \text{softmax}(S_{ij})
]

This gives attention weights.

Step 3: Weighted sum of values

Each token collects information:

[
c_i = \sum_j \alpha_{ij} v_j
]

This produces the context vector for token (i).

Why it is the key output

1. It is the “information fusion” step

Self-attention is fundamentally about:

mixing information across tokens

The context vector is where this mixing becomes explicit.

2. It replaces the original token representation

Before attention:

  • each token is isolated embedding

After attention:

  • each token becomes a contextual embedding

So:

context vector = new representation of the token in context

3. It carries long-range dependencies

Unlike raw embeddings:

  • context vectors incorporate distant tokens

  • even tokens far apart can influence each other

This is what gives transformers their power.

4. It is what feeds the rest of the model

The context vector is passed to:

  • feedforward layers

  • next transformer layers

  • final prediction head (after stacking layers)

So it is the fundamental working representation inside the model.

Key intuition

Self-attention does not directly output predictions.

Instead, it produces:

a richer version of each token that “knows about” the entire sequence

That enriched representation is the context vector.

Imagine each person in a group discussion:

So each person leaves the discussion with:

a new understanding shaped by the group

The context vector is the output of self-attention because it represents:

the transformed token embedding after aggregating information from all other tokens using learned attention weights

It is the core mechanism that allows transformers to build context-aware representations.

Why is dot product (not cosine or Euclidean) used as similarity in attention?

Definition

In attention, similarity between tokens is computed using the dot product between their query and key vectors
[
\text{score}(i,j) = q_i \cdot k_j
]
This score determines how much one token attends to another

Why dot product is used

1. Very fast and efficient

Dot product is just multiply and add operations
It can be computed using matrix multiplication, which is highly optimized on GPUs
This makes it scalable for large models

2. Works naturally with learned representations

Queries and keys are learned by the model
The model shapes them so that dot product directly reflects similarity
So we don’t need a handcrafted similarity function

3. Keeps magnitude information

Dot product captures both

  • direction (alignment)

  • magnitude (strength)

Magnitude can represent importance
Cosine similarity removes this information

4. Fits well with softmax

Attention uses
[
\text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)
]
Dot product produces raw scores that work cleanly with softmax
Scaling keeps values stable

Why not cosine similarity

Cosine similarity normalizes vectors
[
\frac{q \cdot k}{|q||k|}
]

Problems

  • extra computation for normalization

  • loses magnitude information

  • slower at scale

Why not Euclidean distance

Euclidean distance measures distance, not similarity
[
|q - k|^2
]

Problems

  • more expensive to compute

  • needs conversion to similarity

  • not as efficient for matrix operations

Key intuition

Dot product answers

  • how aligned are the vectors

  • how strong is that alignment

This is exactly what attention needs

Short interview-ready answer

Dot product is used in attention because it is simple, efficient, and works well with learned embeddings. It captures both alignment and magnitude between vectors, helping the model measure relevance effectively. Compared to cosine similarity and Euclidean distance, it is faster to compute at scale and integrates naturally with softmax, making it ideal for transformer models.

Why is softmax used to normalize attention scores instead of dividing by their sum?

Definition

In attention, raw scores from (QK^T) are converted into weights using softmax:

[
\alpha_{ij} = \frac{e^{s_{ij}}}{\sum_k e^{s_{ik}}}
]

This turns scores into a valid probability distribution.

Why not simple division

A naive approach would be:

[
\alpha_{ij} = \frac{s_{ij}}{\sum_k s_{ik}}
]

But this has several problems.

Key reasons softmax is preferred

1. Handles negative values correctly

Raw attention scores can be negative
With simple division

  • negative values lead to negative weights

  • weights no longer behave like probabilities

Softmax fixes this by using exponentials

  • all outputs become positive

2. Produces a proper probability distribution

Softmax ensures

  • all weights are positive

  • all weights sum to 1

Simple division does not guarantee this if scores are negative or mixed.

3. Emphasizes important tokens

Softmax amplifies differences

  • large scores become much larger after exponentiation

  • small scores shrink toward zero

This creates sharper, more meaningful attention

Simple division keeps values more uniform

  • weaker distinction between important and unimportant tokens

4. Stable and smooth gradients

Softmax is smooth and differentiable

  • works well with gradient-based optimization

Simple division can behave poorly when

  • sum of scores is close to zero

  • values fluctuate widely

5. Works well with scaling

Attention uses

[
\text{softmax}\left(\frac{QK^T}{\sqrt{d}}\right)
]

Softmax combines well with scaling to control sharpness
Simple division does not provide this flexibility

Key intuition

Softmax acts like a soft selection mechanism

  • highlights the most relevant tokens

  • suppresses less relevant ones

Simple division just rescales numbers

  • it doesn’t create focus

Short interview-ready answer

Softmax is preferred over simple division because it converts attention scores into a valid probability distribution with positive values that sum to one, even when scores are negative. It also amplifies important differences between scores, allowing the model to focus more sharply on relevant tokens. In contrast, simple division can produce negative or unstable weights and does not emphasize important tokens effectively, making it unsuitable for attention.

The sequence is passed through multiple transformer blocks, each consisting of layer normalization, multi-head self-attention (QKV projection, attention scoring, masking, softmax, weighted sum, and output projection), followed by a feedforward network with residual connections and dropout. After all layers, a final layer normalization is applied, and the output is projected to vocabulary logits using a linear layer. These logits represent the model’s predictions for the next token at each position.

Walk through autoregressive text generation step by step, from first token to stop?

Definition

Autoregressive text generation is the process where a model generates text one token at a time, each time conditioning on all previously generated tokens

Step-by-step process

1 Initialize input

Start with a prompt
Example
"Once upon a time"

Tokenizer converts it to token IDs

2 First forward pass (prefill)

  • Entire prompt is fed into the model

  • Model processes all tokens in parallel

  • Produces logits for each position

We only care about the last position

3 Convert logits to probabilities

Apply softmax to last-token logits

[
P = \text{softmax}(\text{logits})
]

This gives probability for every token in vocabulary

4 Choose next token

Select next token using a decoding strategy

  • greedy (argmax)

  • sampling

  • top-k / top-p

Example
"Once upon a time" → next token = "there"

5 Append token

Add the new token to the sequence

Sequence becomes
"Once upon a time there"

6 Repeat forward pass (decode phase)

  • Feed updated sequence into model

  • In practice, reuse past computations via KV cache

  • Compute logits only for the new position

7 Generate next token

Again

  • apply softmax

  • sample or select token

  • append to sequence

Repeat this loop

8 Continue generation

Each step

  • model sees all previous tokens

  • predicts next token

Sequence grows one token at a time

9 Stop condition

Generation stops when one of the following occurs

  • special end-of-sequence token is generated

  • maximum token limit reached

  • custom stopping rule (e.g., newline, keyword)

Key intuition

At each step
the model answers

given everything so far, what comes next

Then uses its own output as input for the next step

Example flow

Prompt
"I love programming because"

Step 1 → "it"
Step 2 → "is"
Step 3 → "fun"
Step 4 → "."

Final output
"I love programming because it is fun."

Short interview-ready answer

Autoregressive generation starts by feeding the prompt into the model to compute logits, then selecting the next token based on probabilities. This token is appended to the sequence, and the process repeats, with the model conditioning on all previously generated tokens. Using techniques like KV caching, only the new token is processed each step. Generation continues until a stop condition such as an end-of-sequence token or maximum length is reached.

Define greedy decoding and discuss its main weaknesses?

Definition

Greedy decoding is a text generation strategy where, at each step, the model selects the token with the highest probability (argmax) from the output distribution

[
\text{next token} = \arg\max P(\text{token})
]

This decision is made independently at every step

How it works

At each generation step

  • compute logits for the next token

  • apply softmax to get probabilities

  • pick the token with the highest probability

  • append it to the sequence

  • repeat

Key characteristic

It always chooses the locally most probable token, without considering future consequences

Main weaknesses

1 Gets stuck in suboptimal sequences

Greedy decoding optimizes locally, not globally

A slightly less probable token now might lead to a better overall sentence later, but greedy never explores that

2 Low diversity

It always produces the same output for the same prompt

No randomness

  • no variation

  • no creativity

3 Repetitive outputs

Greedy decoding often leads to repetition

Example

  • repeated phrases

  • loops like "the the the"

Because high-probability tokens keep reinforcing themselves

4 Poor handling of ambiguity

If multiple valid continuations exist

  • greedy picks only one path

  • ignores other plausible options

5 Error accumulation

Once a wrong token is chosen

  • all future predictions depend on it

  • cannot recover

Key intuition

Greedy decoding is like
always picking the best immediate option without thinking ahead

When it works well

  • tasks requiring deterministic output

  • short or simple completions

  • when highest-probability path is clearly correct

Short interview-ready answer

Greedy decoding is a strategy where the model selects the highest-probability token at each step. While simple and fast, its main weaknesses are that it makes only local decisions, lacks diversity, often produces repetitive or dull outputs, and can get stuck in suboptimal sequences because it does not explore alternative possibilities.
